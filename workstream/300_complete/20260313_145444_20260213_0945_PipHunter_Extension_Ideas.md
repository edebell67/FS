## Objectives
- Expand to additional asset classes: crypto, futures, options.
- Explore arbitrage opportunities within existing data structures.

## 10 Extension Ideas

1. Signal Route Attribution Cards
- Add route metadata to consumer signals: `breakout`, `grid_live`, `trade_bucket`, `psb_family`.
- Use existing fields (`source_screen`, `source`, `trade_block_reason`) for transparency and trust.

2. Execution Probability Layer
- Add a score that predicts whether a signal is likely to execute soon.
- Derive from recent block reasons (e.g. not monitored in grid, side mismatch, gate fail).

3. Unified Multi-Asset Schema (Non-breaking)
- Extend current payload with `asset_class` (`fx`, `crypto`, `futures`, `options`) and `contract_id`.
- Keep core fields (`entry_price`, `current_price`, `net_return`, `status`) unchanged.

4. Source Adapters for Crypto/Futures/Options
- Implement adapter layer to normalize external symbols/quotes into current strategy and trade JSON shape.
- Reuse existing API bridge and signal transformation pipeline.

5. Derived Arbitrage Feed
- Build “arb candidates” from current quote/snapshot streams.
- Start with synthetic/cross-consistency checks and spread dislocations; publish as separate signal type.

6. Family-Based Consumer Products
- Productize existing strategy families (`Fxxx_side`) as app-level bundles.
- Show expected profile bands (avg profit/loss/profitability ranges) rather than single-point stats.

7. Regime-Aware Signal Playlists
- Auto-select strategy families by market regime (trend/range/reversal).
- Map trend -> breakout, reversal -> `_Rev/_R_Rev`, range -> scalper-focused groups.

8. Gamification on Quality, Not Activity
- New challenges tied to execution quality:
  - “Follow only positive net + side_net signals”
  - “Avoid blocked-route signals”
- Uses fields already present in open trade JSON.

9. Marketing Explainability Generator
- Auto-generate one-line “why selected” snippets for each promoted signal.
- Example factors: family rank, side alignment, net/side_net gate pass, observed-day stability.

10. Pro Tier: Multi-Asset + Arbitrage
- Free tier: top FX signals.
- Pro tier: arbitrage alerts + crypto/futures/options feeds + family confidence bands.
- Aligns directly with existing auth/premium model from marketing plan schema.

## Practical First Steps
1. Add schema extensions (`asset_class`, `contract_id`, `route`) in API output without breaking current consumers.
2. Implement execution-probability and route-attribution first (highest UX clarity gain).
3. Add arbitrage as a parallel feed before multi-asset execution routing.

## Notes
- Ideas are intentionally designed to reuse existing `breakout/fs` data structures and JSON conventions.
- No mandatory re-platforming is required for MVP expansion.
