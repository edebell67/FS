# Plan - Auto-Refresh for Live Orders

This plan describes the implementation of an auto-refresh feature on the "Live Orders" page.

## 1. Understanding of Requirements
- **Auto-Refresh Toggle**: Add a checkbox or switch to enable/disable auto-refresh.
- **Refresh Frequency**: Implement a fixed (e.g., 30s) or selectable interval.
- **Visual Feedback**: Show a countdown or indicator when active.
- **Versioning**: Update to `V20260107_1330`.

## 2. Plan of Approach

### UI Layer (`TradeApps\breakout\live_trades.html`)
1.  **Controls**:
    - Add a container in the header for "Auto Refresh".
    - Add a `<select>` for interval (Off, 5s, 10s, 30s, 60s).
2.  **Logic**:
    - Use `setInterval` to call `loadLiveTrades()`.
    - Clear the interval when "Off" is selected or when the date is changed to a past date (to avoid unnecessary background polling on historical data).
    - Ensure refresh doesn't trigger if the page is currently in the middle of a fetch.

### Versioning
1.  Update `constants.py` and footer of `live_trades.html` to `V20260107_1330`.

## 3. List of Changes
- [ ] Add Auto-Refresh dropdown to header.
- [ ] Implement `setInterval` logic in scripts.
- [ ] Add safety check: only auto-refresh if the selected date is "Today".

## 4. Verification Plan

- [ ] Select "Off" and verify it stops.
- [ ] Change date to yesterday and verify it turns off or stops auto-refreshing.
