# Plan: Convert Card Save to Trade Bucket and Fix Metadata Leakage

## 1. Understanding of Requirements
- The user wants the "Save" button on each chart card in `multi_chart.js`, `multi_chart_v2.js`, and `multi_chart_v3.js` to save that specific group of strategies as a "Trade Bucket".
- The bucket name should follow the format: `{mmdd_hhmm}_{product with highest net}_{strategy}_{strategy params}`.
- Metadata leakage fix (from previous plan) still needs to be applied to `common.py`.
- Update version in `constants.py`.

## 2. Plan of Approach

### A. UI Changes (Multi-Chart JS files)
1.  **Modify `createGroupCard`**:
    *   Change the `onclick` of the `.action-btn.save` button to `saveToBucket('${groupName}')`.
    *   Update title to "Save to Trade Bucket".
2.  **Implement `saveToBucket(groupName)`**:
    *   Get `activeOverlays` for the group.
    *   Calculate the "Leader" (strategy with highest net/delta) at the current playback time.
    *   Parse the leader's key to get `product`, `strategy_name`, and `params`.
    *   Construct name: `MMDD_HHMM_PRODUCT_STRAT_PARAMS`.
    *   Call `POST /api/trade_buckets` with the group's strategies.

### B. Backend Changes (`common.py`)
1.  **Modify `_trigger_grid_live_activation`**:
    *   Delay setting metadata (`sent_at`, `source_screen`, etc.) until AFTER successful order creation.
    *   If promotion is blocked by `max_live_trades`, do not persist these fields in the JSON.

### C. Version Update
1.  Update `fs/constants.py` to `V20260204_1255`.
