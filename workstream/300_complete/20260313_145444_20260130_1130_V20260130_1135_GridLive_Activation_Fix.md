# Plan: Fix Grid-Live Activation for Incoming Trades

**Version:** V20260130_1135
**Date:** 2026-01-30 11:35

## 1. Understanding of Requirements
The user is reporting that `is_live_trade` is not being set to `true` for incoming trades even when they match the entries in `grid_live.json`. 

Investigation revealed two primary issues:
1.  **Parser Bug**: `_load_grid_live_map` in `common.py` incorrectly expects `grid_live.json` to be a dictionary of lists, but it is currently saved as a flat list by the UI/API. This causes an `AttributeError` (or failure to parse), resulting in an empty mapping.
2.  **P&L Guard**: `_trigger_grid_live_activation` contains a check `if effective_value <= 0: return False`. For new trades, the initial P&L is almost always negative due to spread and commission. This blocks "immediate" activation from the Grid Live monitoring even if requested in the configuration.

## 2. Plan of Approach
1.  **Update Parser**: Modify `_load_grid_live_map` in `common.py` to correctly handle both the flat list format and the legacy dictionary format for `grid_live.json`.
2.  **Relax P&L Guard**: Modify `_trigger_grid_live_activation` in `common.py` to bypass the P&L guard if `bypass_criteria_check` is set to `"immediately"` in the configuration. This ensures that trades intended for immediate monitoried live execution are activated even with the initial minor negative P&L.
3.  **Update Version**: Change the version in `constants.py` to `V20260130_1135`.
