# Plan: Multi-Chart Performance Optimization

Date: 2026-01-25
Version: V20260125_0530

## 1. Objective
Fix the slow loading of charts in Multi-Chart V2 and resolve the "single-trade" display issue by optimizing the SQL backend and API interactions.

## 2. Plan of Approach
1.  **Refactor SQL View (`vwCombined_trades_closed_output_top200`)**:
    *   **Old Problem**: The view was calculating rolling P&L and ranking for every single row in the 600,000+ record archive before filtering.
    *   **New Solution**: Use a tiered CTE approach:
        *   Aggregate final daily P&L per model first (Pass 1).
        *   Rank the models by daily totals (Pass 2).
        *   Perform expensive rolling P&L calculations **only** for the Top 200 "Winner" models (Final Pass).
    *   This eliminates 99.9% of the redundant calculations.
2.  **Optimize API Data Transfer**:
    *   Update `multi_chart_v2.js` to request a much larger page size (`page_size=5000`).
    *   The default of 100 rows was causing charts to be truncated or requiring numerous hidden requests.
3.  **Client-Side Throttling**:
    *   Add a visual "Loading" overlay to indicate data is being fetched.
