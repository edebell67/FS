# Add Rank (Score) Column to Frequency Explorer

## 1. Requirement
The user wants to see a "Rank (Score)" column in the Snapshot History drill-down table.
- This new rank compares the strategy's consistency (Weighted Score) against others at that specific snapshot time.
- It sits next to the existing "Rank" (Net P&L Rank) to allow comparing short-term performance (Net) vs long-term consistency (Score).

## 2. Plan of Approach
1.  **Modify `weighted_race.py`**:
    *   Refactor the race loop: Update the weighted scores *before* generating the snapshot data for that interval.
    *   Calculate the `score_rank` for every strategy based on the updated weighted scores.
    *   Add `score_rank` and `running_score` to the `leaders` list in the snapshot JSON.
2.  **Modify `frequency_explorer.html`**:
    *   Update the Drilldown Modal table header to include "Rank (Score)".
    *   Update `renderDrilldownTable` (or equivalent) to display `d.score_rank`.
3.  **Update Version**:
    *   Update `constants.py` to `V20260203_1200`.
