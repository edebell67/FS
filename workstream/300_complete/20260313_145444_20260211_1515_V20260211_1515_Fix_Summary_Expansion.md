# Plan: Fix Performance Summary Breakdown Expansion State

This document outlines the plan to fix the issue where the Performance Summary Breakdown list auto-expands (or reverts state) upon auto-refresh.

## 1. Understanding of Requirements

The user reported that the "Performance Summary Breakdown" list structure is not consistent during auto-refresh. Specifically, it "auto expands" the list, likely reverting to a default or previous state instead of maintaining the user's manual expansion/collapse choices. This is particularly noticeable after using "Expand All" or "Collapse All" buttons.

## 2. Plan of Approach

The expansion state is tracked in a global Set called `expandedRowsState`. The `renderSummaryGrid` function uses this set to determine which rows to render as expanded during a refresh.
However, the `expandAllHier` and `collapseAllHier` functions currently only manipulate the DOM classes (`expanded`, `hidden-row`) but **do not update the `expandedRowsState` Set**.
Therefore, when an auto-refresh triggers a re-render, the grid is rebuilt based on the stale `expandedRowsState`, causing the UI to revert to the previous state (often fully expanded due to default initialization).

The fix is to ensure `expandAllHier` and `collapseAllHier` correctly add or remove items from `expandedRowsState`.

## 3. List of Changes

*   **`fs/strategy_performance.html`**:
    *   [x] Locate `window.expandAllHier` function within `renderSummaryGrid`.
    *   [x] Update `expandAllHier` to iterate through rows and **add** their `dataset.stableId` to `expandedRowsState`.
    *   [x] Locate `window.collapseAllHier` function within `renderSummaryGrid`.
    *   [x] Update `collapseAllHier` to iterate through rows and **delete** their `dataset.stableId` from `expandedRowsState`.
