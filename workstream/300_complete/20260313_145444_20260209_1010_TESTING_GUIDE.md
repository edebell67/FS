## Test Scenario 1: Manual Testing with Browser

### Step 1: Open the Dashboard
1. Navigate to: `http://localhost:5000/strategy_performance.html`
2. Open browser console (F12) to see logs
3. You should see: `[BIAS-MONITOR] Started checking for bias flips every 5 seconds`

### Step 2: Verify Initial State
- Console should show: `[INITIAL BIAS] BUY - No action taken`
- No modal should appear (bias hasn't changed yet)

### Step 3: Simulate a Bias Flip
To test the modal, you need to trigger a bias change. You can:

**Option A: Wait for natural bias flip**
- Keep the page open
- Wait for `automated_strategy_picker.py` to detect a real bias change
- This could take minutes/hours depending on market activity

**Option B: Manual simulation (recommended for testing)**
1. Open `json/live/2026-02-09/_targeted_strategies.json`
2. Change `"bias": "BUY"` to `"bias": "SELL"`
3. Save the file
4. Within 5 seconds, the page will detect the change
5. Modal should appear with:
   - Bias change: BUY → SELL
   - Recommended strategy (best scalper for SELL)
   - 4 action buttons

### Step 4: Test Each Button

**Button 1: ✅ Activate Strategy**
- Click "Activate Strategy"
- Should add strategy to `grid_live.json`
- Should log action to `notification_log.json`
- Alert: "Strategy activated: [strategy] on [product]"

**Button 2: ❌ Dismiss**
- Click "Dismiss"
- Modal closes
- Action logged as "dismissed"
- No changes to `grid_live.json`

**Button 3: ⚡ Auto-Activate (No More Prompts)**
- Click "Auto-Activate"
- Activates strategy immediately
- Sets `ai_picker_prompt: false` in `config.json`
- Future bias flips will auto-activate without showing modal
- Alert: "Auto-activate mode enabled..."

**Button 4: 🔕 Suspend All Notifications**
- Click "Suspend"
- Sets `bias_notifications_suspended: true` in `config.json`
- Future bias flips will be ignored
- Alert: "Bias-flip notifications suspended..."

### Step 5: View Notification Log
1. Click the "NOTIFICATION LOG" button in the header
2. Should show a table with all logged actions
3. Columns: Timestamp, Bias Change, Strategy, Product, Action

## Troubleshooting

**Modal doesn't appear:**
- Check console for errors
- Verify `_targeted_strategies.json` exists
- Verify bias actually changed
- Check `ai_picker_prompt` is `true`
- Check `bias_notifications_suspended` is not `true`

**Actions not logging:**
- Check Flask server is running
- Check `/api/notification_action` endpoint
- Check file permissions on `notification_log.json`

**Strategy not activating:**
- Check `/api/smart_target/activate` endpoint
- Check `grid_live.json` file permissions
- Check Flask server logs for errors
