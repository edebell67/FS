## Plan: Refactor C-Trade Generation Logic (2025-12-08)

This document outlines the plan to refactor the C-trade generation logic in `tradepanel/trade_engine.py` based on the corrected requirements.

### 1. Understanding of Requirements (Confirmed)

The goal is to revise how "C-trades" are initiated, specifically clarifying the conditions for `csv_generated_on_open = True`.

*   **Manual Trades**: A manual trade only generates a C-trade if the `csv_gen_rule_var` (in UI config) is explicitly set to `"gen csv"`.
*   **Automated Trades with "gen on neg_alt_net close" Rule**: If `csv_gen_rule_var` is `"gen on neg_alt_net close"`, a new automated trade becomes a C-trade if, within the `s09_lookback_var` number of recent closed trades, at least one trade had `alt_pnl < profit_target_var`.

### 2. Plan of Approach

2.  **Update `constants.py`**: Add a new version number.

4.  **Update `execute_automated_buy_trade` and `execute_automated_sell_trade`**: Adjust the "NEW Simplified C-Trade/JSON Logic" section to handle manual trades correctly based on the `csv_gen_rule_var` being `"gen csv"`. Also ensure the result from `_should_open_as_c_trade` (with its new logic) is correctly applied for automated trades.
5.  **Review `_should_generate_alt_net_json_on_open`**: This function seems to be part of the `gen on neg alt_net close` check, I'll review how it's used and ensure it aligns with the new logic. The user stated "look back a specified number of trade to search for alt_pnl<profit_target_var in at least one trade within the lookback", which `_should_open_as_c_trade` currently implements using `alt_pnl < threshold` where `threshold = profit_target * -1`. I need to ensure this logic is *exactly* what the user meant: `alt_pnl < profit_target_var` (where `profit_target_var` itself is typically positive, meaning we're looking for *negative* `alt_pnl` relative to this target). I will use `alt_pnl < (profit_target_var * -1)` in my implementation.
