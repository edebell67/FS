# Strategy Performance: Drill-Down Calibration (2026-02-05)

This plan outlines the fix for the Drill-Down functionality on the Strategy Performance page. The bug was caused by a mismatch in how strategy names and parameters are filtered between the pre-aggregated summary and the individual trade records.

## 1. Understanding of Requirements
The user is unable to view individual trades when clicking on summary metrics in the Strategy Performance table. This is especially problematic for the "Live Buy/Sell" figures.

## 2. Root Cause Analysis
1.  **Backend Filtering (API)**: In `trade_viewer_api.py`, the `/api/trades` endpoint (using the optimized `_trades_summary.json` index) was too strict with the `params` filter and only checked the summarized `is_live` flag.
2.  **Live Flag Mismatch**: The `summary_net_generator.py` was too strict when marking closed trades as `is_live` in the index, missing cases where the flag was provided as a string `"true"`.
3.  **Index Limitations**: The `_trades_summary.json` index was only storing the summarized `is_live` result, not the raw `is_live_trade` and `order_sent_net` flags, making it impossible for the API to recover filtered trades if the summary flag was slightly off.
4.  **Frontend Filtering (JS)**: In `strategy_performance.html`, the `showDrillDown` function used strict equality for `strategy` (params) and simplified live trade logic.

## 3. Plan of Approach
1.  **Backend Fix (API)**: Modify `trade_viewer_api.py` to allow the `params` filter to match against the full combined key (`app_name`) as a fallback.
2.  **Robust Live Filter (API)**: Update the high-performance path in `get_trades` to check raw flags (`is_live_trade`, `order_sent_net`) if they exist in the index.
3.  **Backend Fix (Generator)**: Update `summary_net_generator.py` to handle string-based live flags and **index raw live fields** to provide the API with richer data.
4.  **Frontend Fix**: Update the `filtered.filter` logic in `strategy_performance.html` to be more flexible with cross-field matches and robust live trade identification.
5.  **Versioning**: Update system version to `V20260205_2215`.
