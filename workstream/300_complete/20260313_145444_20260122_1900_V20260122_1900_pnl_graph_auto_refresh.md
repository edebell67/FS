# Plan: Add Auto-Refresh to P&L Graph Panel

This plan outlines the changes to add an auto-refresh feature to the `pnl_graph.html` page, allowing users to toggle periodic data fetching.

## 1. Understanding of Requirements
The goal is to implement an "Auto Refresh" feature on the Multi-Overlay P&L Graph page. This should:
- Provide a toggle in the UI to enable/disable auto-refresh.
- Refresh the data every 5 seconds when enabled.
- Pause auto-refresh during playback simulation to avoid interference.
- Ensure the manual "Refresh" button actually fetches new data from the server.

## 2. List of Changes

### `pnl_graph.html`
- [x] Add an "Auto" checkbox and label next to the "Refresh" button in the Global Configuration section.

### `pnl_graph.js`
- [x] Add `autoRefreshEnabled` state (defaulting to `true`).
- [x] Implement `initAutoRefresh()` to manage the 5-second `setInterval` for `fetchData`.
- [x] Add an event listener for the `autoRefreshCheck` checkbox to toggle the state.
- [x] Update `togglePlayback()` to clear the refresh interval when playback starts.
- [x] Update `togglePlayback()` (stop block) to resume the refresh interval if playback stops manually.
- [x] Update `playbackTick()` to resume the refresh interval when playback reaching the end of the day.
- [x] Enhance `displayCurrentSelection()` to call `fetchData()` before applying ranking, ensuring the manual Refresh button is effective.
- [x] Replace the hardcoded `setInterval` at the end of the file with `initAutoRefresh()`.

### `Constants.py`
- [x] Update version to `V20260122_1900`.

## 3. Verification
- Verify the "Auto" checkbox appears in the UI.
- Verify data refreshes every 5 seconds (visible in "Last Update" timestamp).
- Verify toggling the checkbox off stops the refresh.
- Verify starting playback stops the refresh.
- Verify playback finishing resumes the refresh.
