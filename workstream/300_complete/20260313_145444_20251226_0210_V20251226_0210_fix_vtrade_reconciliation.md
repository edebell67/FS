# Plan: Fix V-Trade Reconciliation to Close Excess Trades

## Problem
The `_manage_virtual_trades` function only reconciles the **first** LONG and **first** SHORT trade it finds. If multiple V-Trades exist (from different strategies or manual creation), the extras remain open indefinitely.

Current code (line 1713, 1754):
```python
open_buy_v_trade = next((t for t in open_v_trades if t.get('direction') == 'LONG'), None)
open_sell_v_trade = next((t for t in open_v_trades if t.get('direction') == 'SHORT'), None)
```

This only gets the FIRST matching trade, ignoring all others.

## Solution
Modify the reconciliation logic to:
1. **Close ALL open LONG trades** except the one for the top BUY product
2. **Close ALL open SHORT trades** except the one for the top SELL product
3. Ensure only 2 V-Trades maximum are ever open (1 LONG, 1 SHORT)

## Implementation Steps
1. Instead of using `next()` to get a single trade, collect ALL trades by direction
2. For LONG trades:
   - If top_buy_product exists and passes threshold:
     - Keep the trade matching top_buy_product (if it exists)
     - Close ALL other LONG trades
     - Create new trade if none matches top_buy_product
   - Else:
     - Close ALL LONG trades
3. Apply same logic for SHORT trades
