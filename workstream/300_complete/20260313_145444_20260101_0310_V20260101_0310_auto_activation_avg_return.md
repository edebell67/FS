# Plan: Add Average Return Criteria to Auto-Activation

This document outlines the plan to add an "Average Return" requirement for strategy auto-activation.

## 1. Understanding of Requirements

*   **Criteria 1**: Total Trade Count $\ge$ `min_auto_trades` (default: 3).
*   **Criteria 2**: Win Rate > `min_auto_win_rate` (default: 75% as per user's latest change).
*   **Criteria 3**: Total P&L > `threshold_y` (default: $32).
*   **New Criteria**: Average Return (Total P&L / Total Trades) > `min_auto_avg_return` (configurable).

## 2. Plan of Approach

1.  **Modify `config.json`**:
    *   Add `"min_auto_avg_return": 10.0` (setting a default of $10 per trade).
2.  **Modify `common.py`**:
    *   In `_perform_auto_activation_check`:
        *   Retrieve `min_auto_avg_return` from configuration.
        *   Update both **Net** and **Alt** candidacy loops to calculate `avg_return = total_pnl / total_count`.
        *   Add `avg_return > min_auto_avg_return` to the filter conditions.
3.  **Modify `trade_viewer.html`**:
    *   Add a new field "Min Avg Return ($)" to the Configuration modal UI so the user can adjust it.
4.  **Update Version**:
    *   Update `VERSION` in `constants.py` to `V20260101_0310`.
