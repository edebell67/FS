# Plan: Frequency Explorer Page (2026-01-05)

This plan outlines the creation of a new "Frequency Explorer" page that displays product/strategy combo frequencies from the database, allows sorting, auto-updates, and provides a manual activation feature.

## 1. Understanding of Requirements
- **Data Source**: `http://127.0.0.1:8000/api/vw_top_one_frequency?db=tradedb_sim2` (Sim) or `tradedb` (Live). Note: My API server runs on port 5000, but these URLs point to 8000. I should probably proxy them or ensure the frontend can fetch them.
- **Display**: Beautiful table/grid showing combinations in descending order of `frequency_count`.
- **Features**:
    - Auto-update (30s interval).
    - Sortable columns.
    - "Manual Activation" button on each row.
- **UI Consistency**: Use existing sidebar and styling themes.

## 2. Plan of Approach
1.  **UI Page (`frequency_explorer.html`)**:
    *   Create a new HTML file with the standardized sidebar.
    *   Implement a "Run Mode" toggle (Live/Sim).
    *   Implement a sortable table with columns: Product, Strategy, Direction, Frequency Count, Total Return, Last Update, Actions.
2.  **API Integration**:
    *   Use `fetch` to get data from the provided URLs (port 8000).
    *   Add a local API endpoint in `trade_viewer_api.py` if needed to proxy or handle the activation logic smoothly, or just use the existing `/api/activations` POST endpoint.
    *   Manual Addition Button logic:
        *   When clicked, it should call `/api/activations` (POST) to add the strategy to the `activations.json`.
        *   It needs to determine if it's `_buy_net`, `_sell_net`, etc. Based on "Direction" column.
3.  **UI Polish**:
    *   Modern glassmorphism look.
    *   Micro-animations for sorting and updates.
    *   "Last updated" timestamp.
4.  **Sidebar Update**:
    *   Add "Frequency" link to all `sidebar.css` layouts (or just the active ones). 
5.  **Version Update**:
    *   Update `constants.py` and footer versions to `V20260105_2220`.
