# 20251222_1300_V20251222_1300_Top3_Trade_Count_Activation

## 1. Understanding of Requirements
The user wants to add a fourth condition to the "Auto Net/Alt Check" logic. Currently, any strategy that meets the profitability threshold can be activated. The new requirement is to limit these auto-activations to only the **top 3 strategies ranked by their total number of closed trades for the day**.

This ranking should be applied independently for 'Net' and 'Alt' modes.

## 2. Plan of Approach
1.  **Modify `_perform_auto_activation_check` in `TradeApps/breakout/common.py`**:
    *   The existing logic already aggregates P&L from today's closed trades. I will extend this aggregation to also count the number of trades for each strategy configuration.
    *   I will create a data structure to hold the P&L sum, trade count, and activation key for each potential candidate.
    *   After aggregating all stats, I will filter this list to include only the strategies that meet the profitability threshold.
    *   I will then sort this filtered list in descending order based on trade count.
    *   Finally, I will take the top 3 from the sorted list and these will be the only strategies that get activated.

2.  **Scope**:
    *   The change will be isolated to the `_perform_auto_activation_check` function.
    *   The ranking will be performed separately for 'Net' and 'Alt' candidates.
