# Plan: Implementation 1.3 - Voice Processing API [V20260219_1530]

**Project**: bizPA (Voice-First Small Trader Capture App)  
**Objective**: Implement the `/api/v1/voice/process` endpoint to handle audio/text input, parse intents, and return structured actions.  
**Version**: V20260219_1530

## Approach
1.  **Endpoint**: Define `POST /api/v1/voice/process`. It will accept `text` (from client-side ASR) or `audio` (for server-side ASR).
2.  **ASR Integration**: Use a placeholder for ASR. If text is provided, skip to NLU.
3.  **NLU (Intent Parsing)**: Implement a rule-based + keyword-based parser for core intents: `capture_receipt`, `capture_invoice`, `capture_payment`, `create_note`.
4.  **Slot Extraction**: Use Regex/String patterns to extract `amount`, `client_name`, and `labels`.
5.  **Response**: Return a JSON object with `intent`, `slots`, `confidence`, and `confirmation_text` (TTS).
6.  **Persistence**: Log every interaction in the `voice_events` table for future model training.

## Verification
- Verify `POST /api/v1/voice/process` returns correct intent for "Capture receipt £42".
- Verify slots are extracted correctly (e.g., amount = 42.00).
- Verify a record is created in the `voice_events` table.
