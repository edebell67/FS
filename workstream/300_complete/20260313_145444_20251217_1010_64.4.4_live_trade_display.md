# Live Trade P&L Visualization

**Date:** 2025-12-17 10:10
**Version:** 64.4.4
**Description:** Display the specific P&L of "Live" trades (orders sent to TWS) next to the total summaries, distinguishing between Net and Alt execution.

## 1. Understanding of Requirements
*   **Goal**: Show `(L: $Value)` next to Net and Alt P&L totals in the UI summary cards.
*   **Precision**: "Live Net P&L" must only count trades where a **Net** order was sent. "Live Alt P&L" must only count trades where an **Alt** order was sent.
*   **Drill-down**: Indicate in the trade list which trades were sent (L).

## 2. Technical Challenge & Solution
*   **Current State**: JSON data only has `is_live_trade` (combined boolean). This causes ambiguity (did we execute Net, Alt, or Both?).
*   **Solution**:
    1.  **Backend**: Update all strategy scripts to store `order_sent_net` and `order_sent_alt` explicitly in the JSON output.
    2.  **Frontend**: Update `trade_viewer.html` to calculate totals based on these specific flags.

## 3. Plan of Approach
1.  **Update Strategy Scripts (`breakout.py`, etc.)**:
    *   Modify `_save_trade_json` to include `order_sent_net` and `order_sent_alt`.
    *   Modify `_finalize_trade_json` to preserve these fields.
2.  **Update `trade_viewer.html`**:
    *   **Summary Logic**:
        *   `Live Net P&L` = Sum(Net P&L) where `order_sent_net == true`.
        *   `Live Alt P&L` = Sum(Alt P&L) where `order_sent_alt == true`.
    *   **Display**: Add `(L: $...)` span to the summary cards.
    *   **Table**: Add "Live" column indicating status (Net/Alt/Both/-).

## 4. List of Changes
*   **Scripts**: `breakout.py`, `breakout_R.py`, `breakout_Rev.py`, `breakout_R_Rev.py`.
*   **UI**: `trade_viewer.html`.
