# Strategy Performance: UI & Operational Hardening (2026-02-05)

This plan outlines the implementation of JSON buttons in the trade drill-down and the 23:00 operational cutoff for trading.

## 1. Understanding of Requirements
1.  **JSON Display**: The "JSON" column in the trade drill-down modal needs to show "View JSON" buttons for all trades, allowing users to inspect raw trade data.
2.  **23:00 Cutoff**: A new operational rule is required: at 23:00 local time, the engine should stop opening new trades and force-close all existing open trades.
3.  **Proper Closure**: Ensure that live trades (L-trades) go through the full closure workflow (generating close orders) during the force-close process.

## 2. Plan of Approach
1.  **Indexer Update**: Modify `summary_net_generator.py` to include the `filename` in the trade index. Without this, the frontend cannot link to the raw JSON files.
2.  **Operational Helper**: Add a `is_eod()` helper to `BaseBreakoutStrategy` in `common.py` that checks if the current time is 23:00 or later.
3.  **Entry Guard**: Update `BaseBreakoutStrategy.enter_trade` to block new entries if `is_eod()` is true.
4.  **Force Closure**: Update `BaseBreakoutStrategy.process_new_tick` to force closure of open trades if `is_eod()` is true.
5.  **Versioning**: Update system version to `V20260205_2300`.
