# Plan: Permanent Fix for Trade Bucket Single [N] rows [V20260311_1345]

This document outlines the hotfix and permanent structural reinforcement applied to resolve the issue where single-row [N] strategies were still bypassing Trade Bucket validation rules.

## 1. Problem Statement
Despite previous validation rules, automated workflows (`TB_workflow` and `Top_X`) were still creating Trade Buckets with only one strategy row using the [N] (Net) metric. This violates the "Minimum 2 strategies/metrics" rule and creates non-diversified buckets.

## 2. Analysis of Faults
1.  **Critical Backend Bug**: `_series_value_at_or_before` (line 543 of `trade_viewer_api.py`) was hardcoded to fetch the `"net"` field, ignoring the requested `field` parameter. This caused Buy/Sell splits to have identical P&L values.
2.  **Insufficient Deduplication**: If a workflow found two models with the same parameters (e.g. `breakout_3` and `breakout_3_rev`), it would add them both as `[N]` rows, passing the `len < 2` check but potentially resulting in redundant or single-looking displays.
3.  **Weak Enforcement in Top_X Workflow**: The `Top_X` workflow did not have the same "emergency split" logic as the primary `TB_workflow`.

## 3. Implemented Fixes
1.  **Fixed `_series_value_at_or_before`**: Correctly use the `field` parameter to fetch `buy_net`, `sell_net`, or `alt_net`.
2.  **Strategic Deduplication**: Added a `Set`-based deduplication step in both major workflows to collapse redundant strategy/product/metric combinations before validation.
3.  **Emergency Forced Split**: If only 1 row remains after deduplication, the system now **FORCE-SPLITS** that strategy into `[B]` (Buy) and `[S]` (Sell) metrics automatically.
4.  **Final Length Guard**: Re-validated the final length check to return a hard error if less than 2 unique rows exist.
