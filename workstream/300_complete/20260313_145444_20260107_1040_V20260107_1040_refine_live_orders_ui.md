# Plan - Refining Live Orders & Fixing Orphaned Trades

This plan addresses performance-related UI refinements and a bug causing multiple "OPEN" trades for the same strategy to appear in the dashboard.

## 1. Understanding of Requirements
- **Fix "3 Open Trades"**: Prevent strategies from opening multiple trades and/or orphaning old open trades on disk.
- **Cleanup Orphaned Trades**: Ensure `_load_persisted_state` handles multiple open trades gracefully by orphaning/closing old ones.
- **Conditional 'Close' Button**: Hide the button for trades that are already closed.
- **Sortable Columns**: Improve UI usability with sortable table headers.
- **Versioning**: Update to `V20260107_1140`.

## 2. Plan of Approach

### Trading Logic (`TradeApps\breakout\common.py`)
1.  Modify `_load_persisted_state`:
    - Iterate through all "OPEN" trades found for the current strategy/product.
    - Collect them into a list.
    - Pick the one with the **highest trade_id** as the active `self.open_trade`.
    - **Crucially**: For all OTHER open trades found, automatically update their status to "CLOSED" with `exit_reason: "STALE_ORPHANED"` and `exit_time: now`. This cleans up "ghost" trades from the UI on script restart.

### Strategy Implementation (`TradeApps\breakout\breakout_R_Rev.py`)
1.  Refactor `check_and_enter`:
    - Replace manual `self.open_trade` dictionary creation with a call to `self.enter_trade()`.
    - Remove redundant `_save_trade_json` calls (now handled by `enter_trade`).
    - Add a safety `if self.open_trade: return` check at the start.

### UI Layer (`TradeApps\breakout\live_trades.html`)
1.  **'Close' Button**: Update `createTableSection` to check if `t.status.toUpperCase() === 'OPEN'`.
2.  **Sorting**: Implement `id="sort-entry-time"`, `id="sort-product"`, etc., and add click listeners to headers in `renderTrades`.

### Versioning
1.  Update `constants.py` and footer of `live_trades.html` to `V20260107_1140`.
