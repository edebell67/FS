# Plan: Batch Chart Generation [V20260205_1230]

This plan outlines the implementation of a batch chart creation feature in the Multi-Chart P&Grid. This will allow users to quickly create multiple chart cards based on selectable combinations of products, strategies, and parameters.

## 1. Requirements
- A button to open a "Batch Chart Creator" modal.
- Multi-selection lists/checkboxes for:
    - Products (multi-select list)
    - Strategies (multi-select list - base names like `breakout_`)
    - Parameter Construction (Checkboxes):
        - Sizes (2, 3, 4)
        - TP Pips (5, 10, 20, etc.)
        - SL Pips (10, 20, 50, etc.)
- Logic to generate unique combinations: `Strategy` + `Size` + `_tp{TP}.0_sl{SL}.0`.
- Hard limit of 100 total cards.
- Professional UI matching the current "glassmorphism" aesthetic.

## 2. UI Changes (`multi_chart.html`)
- Add a "Batch Create" button in the "Add Chart" section or "Smart Actions" area.
- Define a new modal `#batchCreateModal` with:
    - Column 1: Multi-select for Product.
    - Column 2: Multi-select for Base Strategy.
    - Column 3: Scrollable checkbox groups for Size, TP, and SL.
    - A "Generate Charts" button showing the total count of cards to be created.
    - A "Clear All" or "Reset" button.
- Add CSS for multi-select and checkbox group styling.

## 3. Frontend Logic (`multi_chart.js`)
- **Modal Initialization**:
    - When the modal opens, fetch unique products, strategies, and parameters from the data already loaded in the application (or fetch from `/api/grid_live`).
    - Populate the selection lists.
- **Selection Handling**:
    - Track selected items for each category.
    - Update a "Preview Count" (Products × Strategies × Params).
- **Generation Logic**:
    - Validate that (count + existing cards) <= 100.
    - Loop through combinations.
    - For each combination:
        - Check if a card with this combination already exists (optional, but good for avoiding dupes).
        - Push new config to `groupConfigs`.
    - Trigger `renderCharts()` and `saveCurrentView()` (if appropriate).
- **Automation**: Update `id="batchCreateModal"` to display results clearly.
