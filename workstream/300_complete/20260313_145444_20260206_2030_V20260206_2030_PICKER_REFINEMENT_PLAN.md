# Plan: AI Picker Bias & Chop Refinement [V20260206_2030]

The goal is to align the AI Picker's BIAS with the daily totals seen on the Performance page, while using recent count ranges to detect "Choppiness" and enforce scalping strategies.

## 1. Requirements

- **Primary Bias**: Based on total trade counts for the entire day (Buy vs Sell).
- **Chop Detection**: If recent (1hr) Buy/Sell counts are within 5% of each other, mark as `CHOPPY`.
- **Strategy Filtering**:
    - If `CHOPPY`: Only allow strategies with TP:SL ratio of at least 1:4 (Scalpers).
    - If `TRENDING`: Allow more variety (Trend + Scalp) to follow the momentum.
- **UI Metrics**: Ensure counts displayed in the header reflect these windows.

## 2. Plan of Approach

1.  **Modify `TradeApps/breakout/fs/automated_strategy_picker.py`**:
    - Calculate `day_buy_count` and `day_sell_count` from all trades found.
    - Determine `day_bias` from these totals.
    - Calculate `recent_count_diff_pct` = `abs(buy - sell) / total`.
    - Set `is_choppy = True` if `recent_count_diff_pct < 0.15`.
    - Update `eligible_apps` loop:
        - Baseline: Must have positive net return on the bias side.
        - If `is_choppy`: Force `tp/sl <= 0.25`.
        - If NOT `is_choppy`: Allow `tp/sl <= 0.5` (Relaxed).
2.  **Update `TradeApps/breakout/fs/constants.py`**:
    - Update version to `V20260206_2030`.
