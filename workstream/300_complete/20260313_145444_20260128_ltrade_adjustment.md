Title: 2026-01-28 L-Trade Trigger Adjustments

Summary
-------
Move the L-trade initiation up into the breakout strategy entry point. Instead of grid_live_monitor.py polling for positive P&L and generating TWS orders, we will:
1. Detect when a newly opened virtual trade is configured as "live" (based on grid_live.json) and flag it immediately.
2. Reuse the existing L-trade file creation helpers to drop the TWS order JSON right when the virtual trade opens.
3. Let grid_live_monitor.py keep its logging/sent-trade tracking logic and continue closing live trades, but stop spawning new ones itself.

Key Points
----------
- **Trigger**: As soon as `_save_trade_json()` writes an entry with `is_live_trade: true`, we call the same helper that grid_live_monitor.py uses today to produce `grid_live_*_open_tradeable.json`.
- **Sizing rules**: Keep pulling `trade_qty_percent` and `max_size` from fs/config.json. Confirm the helper already adheres to those settings and avoid duplicating sizing code.
- **File naming**: Update the helper to use the GUID-based format (`grid_live_{guid}_{...}.json`) so strategy-triggered orders match the existing convention.
- **Closing**: When the virtual trade closes, the normal `_cl.json` rename still happens. grid_live_monitor.py watches for those closures and emits matching `_close_tradeable.json` files, so the L-trade lifecycle remains intact.
- **UI toggle impact**: The multi_chart toggle still edits grid_live.json, but its only effect now is whether the strategy marks trades as live. The monitor no longer auto-creates orders when toggles are ON.
- **Scope**: Minimal code churn — mostly hooking into the entry path, sharing helper functions, and removing the redundant trigger loop from grid_live_monitor.py.

Next Steps
----------
1. Update BaseBreakoutStrategy (or a shared mixin) to check grid_live.json when a trade opens, set `is_live_trade`, and call the L-trade helper if needed.
2. Refactor the L-trade helper from grid_live_monitor.py into a shared module so both the strategies and the monitor use the same order-generation code.
3. Remove/disable the “trigger open L-trade” branch from grid_live_monitor.py, leaving it to handle status/logging and closings.
