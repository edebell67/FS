# Plan: Speed up Strategy Performance Drill-downs

## 1. Requirement
The user reported that drill-down trade loading in the Strategy Performance Stats screen is slow. We need to implement caching and optimization to ensure fast response times for subsequent loads.

## 2. Approach
1.  **Frontend Caching** (`strategy_performance.html`):
    - Introduce a global `tradeCache` (a Map) to store trade results.
    - The cache key will be a string combination of `mode`, `date`, `product`, `strategy`, and `params`.
    - Update `showDrillDown()` to:
        - Check the cache first.
        - Only fetch from `/api/trades` if the data is not in the cache.
        - Store the result in the cache upon success.
    - Clear the cache whenever the `Run Mode` or `Date` picker changes.

2.  **Backend Verification** (`trade_viewer_api.py`):
    - Check the logic in `get_trades()` to ensure it prioritizes the optimized `_trades_summary.json` path.
    - Ensure that the summarized index is actually being hit for GBPNZD and other filtered requests.

3.  **UI Enhancements**:
    - Add a "Cache Hit" indicator in the terminal or console for debugging.
    - Ensure the "Loading trades..." message is only shown when a real fetch is happening.

4.  **Version Update**:
    - Increment version to `V20260312_1445`.
