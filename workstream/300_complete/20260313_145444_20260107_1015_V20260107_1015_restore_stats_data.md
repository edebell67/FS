# Plan - Restore Strategy Performance Data visibility

My recent API performance optimization for the Live Orders page incorrectly filtered out all non-live trades at the server level. While this was great for the Live Orders page, it broke the Strategy Performance page which relies on seeing all trades (including simulation/virtual ones) to calculate aggregate stats.

## 1. Understanding of Requirements
- Restore full trade data visibility for the Strategy Performance page.
- Maintain optimization benefits for the Live Orders page.
- Ensure only "Live" trades are shown on the Live Orders page if requested.

## 2. Plan of Approach

### API Layer (`trade_viewer_api.py`)
1.  Add a `live_only` query parameter to `/api/trades` (defaults to `false`).
2.  If `live_only=true`, apply the restrictive filter. Otherwise, return all trades (up to the limit).
3.  Increase the default `limit` to `1000000` so the Performance page isn't truncated by default.

### UI Layer (`live_trades.html`)
1.  Update the `fetch` call to explicitly use `live_only=true` and a reasonable `limit=2000` to keep the page snappy.

### Versioning
1.  Update version to `V20260107_1015`.
