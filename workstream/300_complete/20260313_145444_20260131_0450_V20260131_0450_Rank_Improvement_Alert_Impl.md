# Rank Improvement Alert & Frequency Race Dual-Mode Update

## 1. Overview
This update implements the "Rank Improvement Alert" system in the Frequency Explorer and enables dual-mode (Live/Sim) support for the background frequency race generator. This ensures users are notified not just when a strategy takes the #1 spot, but also when it makes significant moves up the leaderboard (into the Top 5).

## 2. Changes Implemented

### Backend
*   **weighted_race.py**:
    *   [x] Refactored `main()` to run a continuous loop processing both `live` and `sim` modes every 5 minutes.
    *   [x] Removed dependency on single-mode config file for run mode.
    *   [x] Added error handling for global loop stability.

### Frontend (frequency_explorer.html)
*   **State Management**:
    *   [x] Added `previousRanks` to track history of strategy positions.
    *   [x] Added `rankImprovementAlertEnabled` flag.
*   **Rank Detection Logic**:
    *   [x] Replaced `checkRank1Change` with `checkRankChanges`.
    *   [x] Implemented logic to detect when a strategy moves UP in rank (lower index) while being in the Top 5.
*   **Alert UI**:
    *   [x] Enhanced `showRankAlert` to support 'improvement' type.
    *   [x] Added Up Arrow icon (`<i class="fas fa-arrow-up"></i>`) and distinct title "Rank Improved!".
    *   [x] Added context details: "Moved from #X to #Y".
    *   [x] Added "Test Alert" (Bell Icon) button to header for on-demand verification.

## 3. Verification
*   [x] **Sim Mode**: Confirmed `weighted_race.py` is generating `_frequency.json` for Sim mode.
*   [x] **Alert Trigger**: Confirmed "Test Alert" button triggers the yellow popup with improvement set details.
*   [x] **Logic**: Confirmed improvement alerts only trigger for Top 5 candidates moving up.
