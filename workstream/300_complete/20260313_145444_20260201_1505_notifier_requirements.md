# Notifier Enhancements

## Requirements
- Provide user-selectable rank thresholds (rank #1, rank #2, below #3, etc.) so the yellow notification only appears for the preferred events instead of every rank improvement.
- Persist the chosen notification preference so it survives closing the banner and reloading the Frequency Explorer.
- Add a suspend mode that suppresses visible notifications but keeps the last preference alive; while suspended, still auto-add rank #1 leaders to `grid_live` (i.e., “suspend but act”), and automatically reinstate the saved preferences when notifications are resumed.

## Plan
1. Surface new controls in `frequency_explorer.html` for selecting the notification threshold and toggling suspend mode; those controls must write through `/api/config`.
2. Update the client-side notification logic so it reads the persisted preferences at startup and respects the suspend flag while still executing the auto grid-live action for rank #1.
3. Ensure the stored settings (rank filter + suspend flag) are honored every time the notification modal appears so the experience feels persistent.
