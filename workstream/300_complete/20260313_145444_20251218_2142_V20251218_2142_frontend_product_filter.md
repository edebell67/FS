# 20251218_2142_V20251218_2142_Frontend_Product_Filter

## 1. Understanding of Requirements
The goal is to add a product selection filter to the `trade_viewer.html` dashboard to support the new multi-product backend capabilities. This allows users to filter trades by product (e.g., GBP, EUR) and ensures that strategy cards are separated by product.

## 2. Plan of Approach

### Frontend (trade_viewer.html)
1. **Product Selection UI**: Add a multi-select dropdown for "Product" in the controls section.
2. **Dynamic Population**: Implement `populateProductFilter(trades)` to detect available products from the loaded trade data and populate the dropdown.
3. **Filtering Logic**: Update `loadTrades` to apply client-side filtering based on selected products.
4. **Grouping Logic**: Update `groupTrades` to use a key that includes the product: `${trade.app_name}_${trade.product}_${trade.strategy}`. This ensures separate cards for different products using the same strategy.
5. **UI Display**: Update `renderSummary` to show the product name on each card.

### API (trade_viewer_api.py)
* Ensure `product` is consistently returned in the trade objects (already handled by `common.py` and served by the API).
