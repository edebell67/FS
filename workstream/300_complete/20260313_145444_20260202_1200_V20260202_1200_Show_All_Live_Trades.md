# Plan: Show All Live Trades in Dashboard

## 1. Understanding of Requirements
The user reported that the "Live Strategy Trades (L-Trades)" screen is not showing all live trades. This is likely due to strict filtering in the `live_trades.html` frontend and `trade_viewer_api.py` backend, which restricts the display to trades explicitly marked as `is_live_trade`, `order_sent_net`, or `order_sent_alt`. 

To resolve this and ensure full visibility of all trade activities in the Live run mode, I will remove these strict filters.

## 2. Plan of Approach
1.  **Modify Frontend (`live_trades.html` in `db` folder)**:
    *   Remove the `live_only=true` parameter from the `.fetch()` call to the `/api/trades` endpoint. This will instruct the backend to return all trades for the specified date and run mode, regardless of their specific "live" flags.
    *   Remove or relax the client-side JavaScript filter `data.trades.filter(...)` that currently hides trades missing the specific flags.

2.  **Versioning**:
    *   Update `Constants.py` to version `V20260202_1200` to reflect this change.

## 3. List of Changes

*   **`TradeApps/breakout/db/live_trades.html`**:
    *   [x] In `loadLiveTrades()`: Remove `&live_only=true` from the API URL.
    *   [x] In `loadLiveTrades()`: Remove strict filtering logic `const liveTrades = data.trades.filter(...)` and instead use `const liveTrades = data.trades;` (or create a pass-through).
    
*   **`TradeApps/breakout/db/constants.py`**:
    *   [x] Update `VERSION` to `V20260202_1200`.

## 4. Validation
*   Verify that `live_trades.html` requests data without `live_only=true`.
*   Verify that `live_trades.html` displays all returned trades in the "Closed Executions" or "Open Positions" sections.
