# Plan for Duplicate Prevention & Activation Dots (2026-01-30)

This plan addresses the requirement to prevent duplicate monitoring entries in `grid_live.json` and to visualize the activation time of strategies on the Multi-Chart Grid.

## 1. Understanding of Requirements
- **Duplicate Prevention**: When adding models to `grid_live.json` through the API, ensure that a `(model, product)` pair does not already exist in another group.
- **Activation Dots**: On the Multi-Chart Grid, display a single distinctive dot on each chart line indicating exactly when that strategy was activated (`activated_at`).

## 2. Plan of Approach

### Backend: `trade_viewer_api.py`
1.  Modify the `update_grid_live` function.
2.  Before adding new models to the `data` list, identify all existing `(model, product)` pairs in groups other than the one currently being updated.
3.  Filter the incoming models to skip any that already exist in other groups.
4.  Apply comments and version tag.

### Frontend: `multi_chart.js`
1.  Update `fetchData` to extract the `activated_at` field from the `/api/grid_live` response and store it in the `activeOverlays` array.
2.  Create a custom Chart.js plugin named `activationDotsPlugin`.
    - This plugin will iterate through datasets.
    - For each dataset matching an overlay with an `activated_at` timestamp, it will calculate the corresponding X/Y position.
    - It will draw a single, large, stroked dot at that position.
3.  Register this plugin in the `Chart` constructor within `updateOrCreateGroupChart`.

### Metadata: `Constants.py`
1.  Update the version number to `V20260130_1515`.
