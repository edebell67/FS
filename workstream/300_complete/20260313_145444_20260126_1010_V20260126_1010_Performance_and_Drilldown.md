# Plan: Performance Optimization & Drilldown Data Enhancement [V20260126_1010]

This plan addresses a significant performance bottleneck in the Multi-Chart V3 dashboard and enhances the Trade Drilldown modal with missing trade details.

## 1. Requirements
*   **Performance**: Identify and resolve slow dashboard loading.
*   **UI Columns**: Add the following columns to the trade drilldown in `multi_chart_v3.html`:
    *   Exit Time
    *   Status (Open/Closed)
    *   L-Trade (Yes/No indicator for live trades)

## 2. Plan of Approach

### A. Performance Optimization (API Level)
*   **Problem**: Every request to `vwCombined_trades_output_top200` was triggering a full SQL Server P&L cache refresh (`sp_refresh_dna_pnl_cache`). This caused significant latency (blocking the API response until the SQL work finished).
*   **Solution**:
    *   Decouple the cache refresh from the API request.
    *   Create a dedicated background service `pnl_cache_refresh_service.py` that refreshes the SQL cache every minute.
    *   Add this service to the `verify_algo_execution_and_restart_02.py` watchdog.

### B. SQL Schema & View Enhancement
*   **View (`vwCombined_trades_output_top200`)**:
    *   Update the view definition to JOIN the optimized `dna_pnl_stream_cache` table with the master `vw_113_Combined_trades_all` view.
    *   This join retrieves `g_close_time` (Exit Time), `trade_status` (Status), and `tradeable` (Live indicator) for each cached trade point.

### C. Frontend Enhancement (`multi_chart_v3.*`)
*   **HTML**: Update modal table headers.
*   **JS**: 
    *   Fetch and store new numeric/text fields in `processedSeries`.
    *   Properly format Exit Time (HH:MM:SS string) and L-Trade (Yes/No).
    *   Ensure column sorting works for the three new columns.
