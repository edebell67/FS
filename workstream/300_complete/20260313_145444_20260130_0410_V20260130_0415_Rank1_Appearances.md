# Plan: Enhancing Frequency Metrics with Rank 1 Appearances

**Version:** V20260130_0415
**Date:** 2026-01-30 04:15

## 1. Understanding of Requirements
The user wants to enhance the "Appearances" metric in the Frequency Explorer. Currently, "Appearances" counts how many times a strategy appeared in the Top 5 of a snapshot. The new requirement is to also show how many times the strategy was specifically at Rank #1. This should be displayed as `Total (+Rank1s)`.

## 2. Plan of Approach
1.  **Track Rank 1s in Backend**: Modify the frequency generation scripts to count how many times each strategy achieves Rank 1 in a snapshot.
2.  **Export New Metric**: Include `rank1_appearances` in the JSON output (`_frequency.json`, etc.).
3.  **Update UI (Leaderboard)**: Update `frequency_explorer.html` to display the rank 1 count next to total appearances.
4.  **Update UI (Consistency)**: Calculate and display rank 1 counts within the "Recent Consistency" logic.
5.  **Maintain Consistency**: Apply these changes to all data source generators (Breakout, DNA Net, DNA Alt).
