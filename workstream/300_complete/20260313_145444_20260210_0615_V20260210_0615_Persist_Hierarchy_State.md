[V20260210_0615] Persistent Hierarchy State
**Current Status**: IMPLEMENTED by Claude Code (2026-02-10 06:15)
**Objective**: Ensure that the hierarchy expansion state (collapsed/expanded) in the Strategy Performance table is persistent across user interactions and data refreshes.

## 1. Understanding of Requirements
The user wants to avoid the frustration of having the hierarchy reset to a default state (fully collapsed or default expanded) whenever they interact with the table (e.g., sorting) or when data updates. The state of each row (Category, Window, Ratio) should be remembered.

## 2. Plan of Approach
1.  **Introduce State Management**: Use a global `Set` (`expandedRowsState`) to store the unique identifiers of expanded rows.
2.  **Stable Identifiers**: Generate stable IDs for each row based on its content (e.g., `Category`, `Category|Window`, `Category|Window|Ratio`) rather than random counters, ensuring state can be mapped back to re-rendered rows.
3.  **Update Rendering Logic**: Modify `renderSummaryGrid` to check the `expandedRowsState` set for each row during generation and apply the `expanded` class and correct visibility (`hidden-row`) based on the state.
4.  **Update Interaction Logic**: Modify `toggleHierRow` to update the `expandedRowsState` set (add/delete) and use a robust, recursive-aware logic (stack-based) to toggle visibility of descendants correctly.
5.  **Initial State**: Pre-populate the state to have Level 1 and Level 2 rows expanded by default on the first load.

## 3. List of Changes
*   **`TradeApps/breakout/fs/strategy_performance.html`**:
    *   [x] **Injection**: Added `expandedRowsState` Set and `isFirstRender` flag.
    *   [x] **Logic Update**: Implemented logic to pre-populate `expandedRowsState` with L1 and L2 keys on first render.
    *   [x] **HTML Generation**: Updated L1, L2, L3, L4 row generation to:
        *   Construct `stableId`s (e.g., `${catKey}|${winKey}`).
        *   Apply `expanded` class if `stableId` is in `expandedRowsState`.
        *   Apply `hidden-row` class based on parent's expansion status.
        *   Add `data-stable-id` attribute to rows.
    *   [x] **Toggle Logic**: Replaced `toggleHierRow` with a new version that:
        *   Updates `expandedRowsState`.
        *   Uses a stack (`visStack`) to correctly manage visibility of nested children based on the expansion state of the entire ancestry chain.

## 4. Verification
*   **Expansion**: Clicking a row toggle correctly expands/collapses children.
*   **Persistence**: (Inferred) Sorting or refreshing the grid will re-render rows. Since `renderSummaryGrid` now uses `expandedRowsState`, the expansion state should be preserved.
*   **Visibility**: Child rows (L2, L3, L4) are only visible if *all* their ancestors are expanded.
