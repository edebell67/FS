# Plan: Single-Canvas Auto Split Chart

## 1. Understanding of Requirements
The user wants the "Split Chart" function to deliver a single-canvas split view (displaying Buy net and Sell net metrics as two separate step-lines overlaid together) instead of rendering two separate tiny canvas panels natively.

## 2. Plan of Approach
1. **Refactor Split/Combo State**: We will modify `getGroupSplitState` and `handleSplitComboAction` in `fs/multi_chart.js`. Instead of toggling a visual `ChartMode.SPLIT` flag, the "Split Chart" button will programmatically remove the singular "Net" overlay and replace it with two new overlays (one for `metric: 'buy_net'` and one for `metric: 'sell_net'`) on the same card.
2. **Combo Reversal**: If a card contains exactly layers for Buy and Sell matching the current strategy, the toggle button state will switch to "Combo Chart". Clicking it will reverse the process, merging them back into a single "Net" overlay.
3. **UI/DOM Cleanup**: Delete obsolete UI elements and legacy code functions that render side-by-side split chart views (`<div class="split-chart-panel">`, `ChartMode`, `renderSplitCharts()`).

## 3. List of Changes
* **`fs/multi_chart.js`**:
  * [ ] Remove `ChartMode`, `groupSplitModes`, `resetSplitModes`, `getGroupSplitMode`, and `setGroupSplitMode` (obsolete).
  * [ ] Rewrite `getGroupSplitState(groupName)` to detect exactly 1 layer `{metric: null}` (Split allowed) vs multiple metric layers including a buy and sell metric (Combo allowed).
  * [ ] Rewrite `handleSplitComboAction(event, groupName)` to modify the `activeOverlays` array:
    * Action "Split": Replace the current layer with 2 layers (`metric: 'buy_net', color: '#10b981'` and `metric: 'sell_net', color: '#ef4444'`).
    * Action "Combo": Strip the buy/sell layers and restore 1 single layer (`metric: null`).
    * Both actions must invoke `updateCharts()`.
  * [ ] Refactor `renderGroupCharts` to always use `renderNetChart()`, removing the `ChartMode.SPLIT` branch.
  * [ ] Delete the `renderSplitCharts` function completely.
  * [ ] Update `createGroupCard()` HTML string to entirely remove the unused `<div class="chart-container split-view">` DOM elements.
