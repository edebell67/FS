# Plan: Fix Strategy Performance Display and Drill-down

This document outlines the plan to fix issues with the Strategy Performance Stats page, specifically the empty drill-down results and the "Summary not found" error.

## 1. Understanding of Requirements
The user reported two main issues:
1.  **Summary not found error**: Displays when the date is not correctly initialized.
2.  **Empty Drill-down**: The modal showing individual trades is empty even when the summary table shows trade counts.
3.  **Filtered mode issue**: Drill-down fails specifically when filters or search terms are applied in the UI.

## 2. Plan of Approach
The fix involves both backend (API) and frontend (HTML/JS) changes to handle missing dates and robustly match strategy names.

### Backend (trade_viewer_api.py):
1.  **Date Normalization**: In `get_stats_summary`, `get_trades`, and `get_virtual_trades`, ensure the `date` parameter defaults to the current UTC date if it's empty or missing.
2.  **Robust Filtering**: In `get_trades`, relax the filtering logic. Instead of requiring an exact match between `strategy_filter` and the internal trade name, allow substring/inclusion matches. Also, remove the restricted glob prefix to ensure all relevant files are scanned.
3.  **Metadata Enrichment**: In `get_stats_summary`, attempt to extract parameters from the strategy name if the `params` field is empty. This will populate the "Parm" column in the UI instead of showing `-`.

### Frontend (strategy_performance.html):
1.  **Date Initialization**: Ensure the date picker is never empty when calling `loadStats` or `showDrillDown`.
2.  **Robust Client-side Filtering**: Update the filter in `showDrillDown` to support substring matches, ensuring that if the API returns trades, they are not accidentally skipped in the UI due to slight name discrepancies.
3.  **Logging**: Add `console.log` statements to help identify where filtering is failing.
