# Plan: Add Cumulative P&L Column to Live Trades Viewer

## 1. Requirement
Add a cumulative P&L column to the "Closed Executions" table in `live_trades.html`. This column should show the running total of Net Return up to that specific trade, respecting the chronological order of trades regardless of the current table sort order.

## 2. Plan of Approach
1.  **Calculate Cumulative P&L**:
    *   In `loadLiveTrades`, create a temporary list of trades sorted by `entry_time` (Ascending).
    *   Iterate through this sorted list and calculate the running sum of `net_return`.
    *   Assign this running sum to a new property `cumulative_net` on each trade object.
2.  **Update UI**:
    *   In `createTableSection`, add a new table header `<th>Cumulative</th>`.
    *   Add a new table data cell `<td>` displaying `t.cumulative_net`.
    *   Apply conditional formatting (positive/negative colors).

## 3. List of Changes
*   **`TradeApps/breakout/fs/live_trades.html`**:
    *   [x] In `loadLiveTrades()`: Implement cumulative calculation logic.
    *   [x] In `createTableSection()`: Add "Cumulative" column header and data cell.

## 4. Verification
*   Verify that the "Closed Executions" table shows the new column.
*   Verify that the value represents the equity curve at the time of the trade (even when sorted by Time Descending).
