# 2026-02-15 V20260215_Midnight Health Alarm Contextual Fix

## 1. Understanding of Requirements
The user wants the fluorescent red health alarm (triggered by `/api/system_health` failures) to be contextual.
*   **Live Mode**: If the system is unhealthy, the red alarm should display.
*   **Sim Mode**: The red alarm should **NEVER** display, regardless of health status.

Currently, `sidebar-loader.js` implements a global health monitor that fires unconditionally. We need to modify it to check the current `runMode` from the UI (e.g., `#runMode` dropdown or similar context) before applying the alarm styles.

## 2. Plan of Approach
1.  **Analyze `sidebar-loader.js` context**:
    *   The script runs globally. We need to identify if there is a reliable way to detect the current "mode" from the page context. Most pages (like `trade_bucket.html`, `trade_viewer.html`) have a `<select id="runMode">` element.
2.  **Modify `sidebar-loader.js`**:
    *   Inside the `checkHealth` function:
        *   Look for the `runMode` element in the DOM.
        *   If `runMode` exists and its value is 'sim', **SKIP** applying the alarm style (or remove it if present).
        *   If `runMode` is missing (some pages might not have it), we might default to showing the alarm or not. Based on "system health", usually implies Live environment issues, so defaulting to Show is safer, but user explicitly wants Sim to be quiet. If we can't determine mode, we should statistically assume Live or look for other indicators.
        *   *Refinement*: Most pages store mode in `#runMode`. If that element is not present, we can check `localStorage` or URL params, but checking the DOM element `#runMode` is the most direct way for the active view.

## 3. List of Changes
*   **`TradeApps/breakout/fs/sidebar-loader.js`**:
    *   [ ] In `checkHealth()`:
        *   Get the value of `document.getElementById('runMode')`.
        *   Add condition: `if (runModeEl && runModeEl.value === 'sim') { removeStyle(); return; }`.
        *   Ensure this check happens *before* applying the red style.

## 4. Verification
*   User will verify by switching between Live and Sim modes on a page with health issues (stale files). Sim should be normal, Live should be red.
