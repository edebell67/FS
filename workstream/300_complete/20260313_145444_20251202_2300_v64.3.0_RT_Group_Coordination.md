# RT Group Multi-Instance Coordination Implementation Plan
**Version:** 64.3.0  
**Date:** 2025-12-02 23:00  
**Objective:** Enable RT group instances to view aggregated metrics and enforce global limits using shared `trade_lifecycle_snapshots` table

## Requirements Summary

For all instances that are members of the RT group:
- Display max open trade quantity limit
- Display current total open quantity across RT group
- Display available quantity to trade
- Display sum of profits (closed trades) across RT group
- Display sum of open profits across RT group
- Display RT membership status indicator

**RT Membership Definition:** Instance is RT member when 'Test CSV' checkbox is **unchecked** (`send_csv_to_test_folder_var = False`)

## Data Flow

```
┌─────────────────────────────────────────────────────────────┐
│ TradePanel Instance (RT Member: Test CSV unchecked)         │
│                                                              │
│  1. Trade Event Occurs (open/close/snapshot)                │
│  2. _send_trade_lifecycle_event() called                    │
│  3. Payload includes: is_rt_member=True, instance_name      │
│  4. POST to /trade_lifecycle_snapshots endpoint             │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ Database: trade_lifecycle_snapshots table                   │
│  - Stores all events from all instances                     │
│  - is_rt_member flag allows filtering RT vs non-RT          │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ Stored Procedure: sp_get_rt_group_metrics                   │
│  - Filters by is_rt_member = 1                              │
│  - Aggregates: SUM(quantity), SUM(pnl), COUNT(*)            │
│  - Returns: current_open_qty, total_profits, open_profits   │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ TradePanel Instance - Background Thread                     │
│  - Periodically calls fetch_rt_group_metrics()              │
│  - Updates self.rt_group_metrics dictionary                 │
│  - Triggers UI refresh                                      │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│ UI: Product Trade Panel - RT Group Metrics Section          │
│  - Reads from manager.rt_group_metrics                      │
│  - Displays: Max/Current/Available Qty, Profits, Status     │
│  - Updates every refresh cycle                              │
└─────────────────────────────────────────────────────────────┘
```

---

## Notes

- **Backward Compatibility**: Existing instances will continue to work. New columns have defaults.
- **Performance**: Aggregation query runs every 10 seconds, minimal DB load.
- **RT Detection**: Based solely on `send_csv_to_test_folder_var` config value.
- **Instance Naming**: Use `app_name_prefix` + `session_guid[:8]` for unique identification.

## V64.3.1 Update (2025-12-03)

**Requirements:**
1.  **'C' Trades Only**: Metrics must only include trades where a CSV was generated (`is_c_trade = 1`).
2.  **Net Quantity**: Opposing 'C' trades (Buy vs Sell) must cancel each other out. `current_open_qty` = `ABS(Total Buy - Total Sell)`.

**Changes:**
- [x] **Schema**: Added `is_c_trade` column to `trade_lifecycle_snapshots`.
- [x] **Backend**: Updated `_send_trade_lifecycle_event` to populate `is_c_trade`.
- [x] **Stored Procedure**: Updated `sp_get_rt_group_metrics` to filter by `is_c_trade` and use Net Quantity logic.
- [x] **Sim2**: Applied all changes to `tradedb_sim2`.
