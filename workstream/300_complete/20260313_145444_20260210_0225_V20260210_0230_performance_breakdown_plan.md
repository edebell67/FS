# Implementation Plan: Strategy Performance Data Breakdown

This plan outlines the addition of a comprehensive data breakdown section to the `strategy_performance.html` dashboard, providing aggregated insights by Category, Window Size, and TP:SL configuration.

## 1. Requirements
- Add a visual summary section at the top of the `strategy_performance.html` page.
- Breakdown should include:
    - **Strategy Categories** (e.g., Breakout, breakout_R, Breakout_Rev, Breakout_R_Rev).
    - **Window Sizes** (2, 3, 4).
    - **TP:SL Configurations** (e.g., 10.0/20.0, 5.0/30.0, etc.).
- Each card should show:
    - #Buys / #Sells counts.
    - Buy Net / Sell Net performance.
    - Total Net for that group.
- Styling must match the premium "Modern Dark" aesthetic of the application.

## 2. Proposed Changes

### 2.1 CSS Integration (`strategy_performance.html`)
- Add styles for:
    - `.summary-grid`: Grid layout for the breakdown cards.
    - `.section-header`: High-contrast headers for different breakdown dimensions.
    - `.dashboard-card`: Premium glassmorphism cards with hover effects.
    - `.dashboard-row`: Flex layout for key-value pairs inside cards.

### 2.2 HTML Structure (`strategy_performance.html`)
- Insert `<div id="summaryGrid" class="summary-grid"></div>` before the main strategy table container.

### 2.3 JavaScript Logic (`strategy_performance.html`)
- **New Function**: `renderSummaryBreakdown(statsData)`
    - This function will process the `statsData` array (which contains product-level aggregations).
    - Use regex/string parsing to extract `windowSize` and `tpSl` from `d.strategy` or `d.parm`.
    - Build three aggregation objects: `catStats`, `windowStats`, and `tpSlStats`.
    - Generate HTML for section headers and cards.
- **Integration**: Update `loadStats()` to call `renderSummaryBreakdown()` after successfully fetching and mapping data.
