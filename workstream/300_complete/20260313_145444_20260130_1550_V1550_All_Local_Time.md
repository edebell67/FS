# Plan for "All Local Time" transition (2026-01-30)

This plan outlines the changes required to move the entire Breakout application away from UTC and transition to exclusive Local Time (based on the server/browser environment).

## 1. Understanding of Requirements
- **Unified Time**: All displays (charts, clocks, logs) should reflect the Local Time of the user/browser.
- **Storage Sync**: Activation times and status updates in JSON files should be stored as Local Time strings without the 'Z' (UTC) indicator, ensuring they are interpreted as Local by standard JavaScript `Date` objects.

## 2. Plan of Approach

### Backend: `common.py` & `trade_viewer_api.py`
1.  Update `common.py` to use `datetime.now()` for `today_str` logic.
2.  Update `trade_viewer_api.py` to use `datetime.now()` for `activated_at` timestamps in `update_grid_live`.

### Backend Utilities: `summary_net_generator.py` & `grid_live_monitor.py`
1.  Replace all `datetime.utcnow()` calls with `datetime.now()`.
2.  Ensure `isoformat()` does not have a 'Z' suffix.

### Frontend: `multi_chart.js`
1.  **Data Processing**: Modify `fetchData` to strip 'Z' from incoming P&L timestamps, forcing Local interpretation.
2.  **Boundaries**: Update `getPlaybackBoundaries` to create `startOfDay` and `endOfDay` using `T00:00:00` (Local) instead of `UTC`.
3.  **Visualization**: Update `activationDotsPlugin` to treat `activated_at` as Local Time.
4.  **UI Display**: Update `updatePlaybackUI` to use `toLocaleTimeString()` for the main clock.

### Metadata: `Constants.py`
1.  Update version to `V20260130_1550`.
