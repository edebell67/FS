# Frequency Explorer Rank Metric Selector Plan

Date: 2026-02-03
Owner: Codex
Location: C:\Users\edebe\eds\TradeApps\breakout\fs\frequency_explorer.html

## Goal
Allow the Rank notifier to use a selectable ranking metric without changing the alert logic flow. The user can switch between:
- Rank (Net): current behavior using final_net/net.
- Rank (Weights): use weighted_score.

The selection must be changeable at any time and persist via config.

## Current Behavior (Baseline)
- Rank alert sorting uses net via getLeaderNet() inside checkRankChanges().
- UI does not expose a rank metric selector.
- config.json is loaded for rank alert preferences and persisted via updateConfig().

## Proposed UX
- Add a selector in the rank alert controls (same panel) labeled "Rank Metric" with options:
  - Net (default)
  - Weighted Score
- Changing the selector updates rank alert behavior immediately (no reload required).
- Persist the selection in config (e.g., rank_alert_rank_metric).

## Data/Config Changes
- New config key: rank_alert_rank_metric
  - Allowed values: "net" | "weighted"
  - Default: "net" when missing or invalid.

## Implementation Steps
1) UI
   - Add a new <select> in the rank alert control area.
   - Example id: rankAlertMetricSelect.
   - Options: Net / Weighted Score.

2) State
   - Add a new JS state variable, e.g. rankAlertMetric = 'net'.
   - Initialize from config.json if present.

3) Rank Metric Getter (Non-invasive)
   - Keep checkRankChanges() structure intact.
   - Replace getLeaderNet usage with a new function that returns the metric based on rankAlertMetric.
     - If rankAlertMetric === 'weighted': return weighted_score (fallback 0).
     - Else: return final_net -> net -> 0 (current behavior).

4) Alert Logic
   - In checkRankChanges():
     - Sort leaders using the new metric getter.
     - Other logic remains unchanged.
   - Ensure the "rank change" tracking uses the chosen metric sorting.

5) Persistence
   - On select change, update rankAlertMetric and call updateConfig({ rank_alert_rank_metric: value }).
   - On initExplorer() config load, set rankAlertMetric from config if present.

6) Optional Alignment (Decision Needed)
   - Summary "Current Front-Runner" currently uses net. Decide whether to:
     - Keep summary fixed to net (no change).
     - Or align summary to selected metric for consistency.
   - If aligning, use the same metric getter for summary top leader.

7) Guardrails
   - Validate config values (fallback to 'net' if unknown).
   - Keep existing behavior if config missing.

## Testing Checklist
- Load page: default rank metric is Net.
- Toggle to Weighted Score:
  - Rank alert top leader changes if weighted_score ordering differs.
  - Rank alert improvements still function.
- Refresh page: selection persists.
- Historical view (if applicable): no errors, selection still applies.
