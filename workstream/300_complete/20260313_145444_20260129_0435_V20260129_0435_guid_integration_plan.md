# Plan: Trade JSON GUID Integration and filename hardening (V20260129_0435)

This plan outlines the steps to integrate a unique GUID into all trades (underlying and live orders) and ensure robust filename handling.

## 1. Requirements Recap
- Add a unique GUID to all new underlying trades.
- Include the GUID in the filename of created JSON orders, before the product name.
- Ensure only one open and one closed trade are linked to the same GUID.

## 2. Plan of Approach

### Core Strategy (common.py)
1. **GUID Generation**: In `BaseBreakoutStrategy.enter_trade`, generate a new GUID using `uuid.uuid4().hex[:8]`.
2. **State Storage**: Add `guid` to the `self.open_trade` dictionary.
3. **Filename Update**: Modify `_generate_trade_filename` and `_compute_trade_filenames` to accept and use this GUID. The underlying trade filename will be: `{script_name}_{guid}_{product}_{ts}_{params}.json`.
4. **Live Orders**: Update `_create_l_trade_order` and `_create_tradeable_json` to propagate the GUID. The order filename will be: `{prefix}_{guid}_{product}_{strategy}_{action}_{ts}_{suffix}.json`.
5. **Persistence**: Ensure `_load_persisted_state` can handle the new filename format.

### Grid Monitor (grid_live_monitor.py)
1. **Order Filenames**: Update `create_l_trade_json` to insert the `trade_guid` into the filename before the product name.

### Versioning
1. Update `Constants.py` to `V20260129_0435`.

## 3. List of Changes

### common.py
- [ ] **Modify `_generate_trade_filename(self, guid: str, current_time: Any)`**: Update to include `guid` before product name.

- [ ] **Modify `enter_trade()`**: Generate GUID, store in `self.open_trade`, and pass to filename computation.
- [ ] **Modify `_create_tradeable_json()`**: Extract `guid` from memory and pass to order creation.

### grid_live_monitor.py
- [ ] **Modify `create_l_trade_json()`**: Update filename template to include `trade_guid` before the product name.

### Constants.py
