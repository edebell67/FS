# UI Refresh Optimization

**Date:** 2025-12-16 23:30
**Version:** 64.4.2
**Description:** Optimize the Trade Viewer auto-refresh to prevent full page reloading/flickering.

## 1. Understanding of Requirements
User requested that auto-refresh should only update "actual data" and not "refresh the whole page". This implies removing the loading spinner during background updates and skipping rendering if data hasn't changed.

## 2. Plan of Approach
1.  **Modify `loadTrades`**: Add `isBackground` parameter.
2.  **Spinner Control**: Only show "Loading..." if `!isBackground`.
3.  **Data Equality**: Compare fetched JSON with specific `allTrades` cache. If identical, return early.
4.  **Error Handling**: Suppress UI error messages during background refresh.
5.  **Auto-Refresh Call**: Update `setInterval` to call `loadTrades(true)`.
