# Plan: Robust State Persistence and Hardware Safety (V20260129_0325)

## 1. Problem Statement
The user identified a discrepancy where 25 JSON files were generated for only 10 expected trades.
The primary cause was **redundant order generation** triggered by two issues:
1. **Filename Fragility**: The strategy scripts were generating corrupted filenames (multi-extensions) and failing to recover their own state on restart, leading to multiple `_op.json` files for the same actual signal.
2. **Monitor Memory Sync**: The monitor was only using its internal `sent_trades.json` to block duplicates. If the GUID changed (due to filename changes or missing IDs), it re-sent the order.

## 2. Plan of Approach

### Phase 1: Robust File Naming (common.py)
Implemented a "Hardware-grade" filename cleaner.
- **Refactor**: `_clean_trade_filename` now recursively strips ALL known suffixes (`.json`, `_op`, `_cl`, `_cld`) ensuring a clean base stem.
- **Single Source of Truth**: All filename methods now derived from this clean stem, preventing nested extensions like `.json_op.json`.

### Phase 2: Enhanced State Recovery (common.py)
Improved the `_load_persisted_state` method.
- **Matching Logic**: The strategy now looks for its existing open trade using both the **Filename Pattern** and the **`script_name` field** within the JSON content.
- **Result**: Even if the script restarts, it will correctly identify and resume its current open trade instead of starting a redundant one.

### Phase 3: Monitor Hardware Safety (grid_live_monitor.py)
Implemented the user's requested "Hardware Safety" check.
- **Disk-Content Check**: Before the monitor creates a TWS order, it now **READS** the underlying trade JSON file.
- **Promotion Block**: If the JSON content shows `is_live_trade: true` (or order sent flags), the monitor **immediately skips** order generation.
- **Self-Healing Sync**: If the monitor finds a trade is already live on disk but missing from its own `sent_trades.json`, it synchronizes its memory to prevent future processing of that trade.
