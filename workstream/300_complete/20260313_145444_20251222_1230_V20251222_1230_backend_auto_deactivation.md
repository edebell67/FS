# 20251222_1230_V20251222_1230_Backend_Auto_Deactivation

## 1. Understanding of Requirements
The user wants to enhance the "Auto Net/Alt Check" logic in the backend. Currently, disabling the master switch (`auto_net_check` or `auto_alt_net_check`) in `config.json` only prevents *new* auto-activations. Existing active strategies remain on.
The requirement is to **automatically deactivate** all currently active strategies for a given mode (Net or Alt) if its master switch is turned off.

## 2. Plan of Approach
1.  **Modify `_perform_auto_activation_check` in `TradeApps/breakout/common.py`**:
    *   Remove the early return check `if not (auto_net_check or auto_alt_net_check): return`. The function must run to handle deactivations even if both flags are false.
    *   Implement **Deactivation Logic**:
        *   Iterate through `activations.json`.
        *   Filter keys that belong to the current process (matching `target_script_names`).
        *   If `auto_net_check` is False: Identify active keys ending in `_net` and queue them for deactivation.
        *   If `auto_alt_net_check` is False: Identify active keys ending in `_alt` and queue them for deactivation.
    *   Implement **Activation Logic** (Conditional):
        *   Only perform the disk scan for P&L stats if at least one flag is True.
        *   If scanning, proceed with the existing logic to auto-activate profitable strategies.
    *   **Save**: Apply all updates (activations and deactivations) to `activations.json`.

2.  **Concurrency Safety**:
    *   Ensure each process only modifies activation keys related to its own strategies (`target_script_names`) to minimize race conditions with other running scripts.
