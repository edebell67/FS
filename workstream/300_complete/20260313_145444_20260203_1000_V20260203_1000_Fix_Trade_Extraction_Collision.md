# Fix Live Trade Extraction Collision

## 1. Requirement
The user reported a missing live trade in the viewer (`06:33:57`, Net 87.5).
Investigation revealed that the trade file exists but `extract_live_trades.py` failed to include it in `_live_trades.json`.
The cause is identified as **Trade ID Collision**: The script uses `trade_id` (e.g., "32") as a dictionary key. Since multiple strategies can generate trades with the same simple integer sequence, later trades overwrite earlier ones in the extraction process.
The fix is to use a composite unique key (e.g., `script_name` + `trade_id`) during the extraction process to ensure all unique trades are preserved.

## 2. Plan of Approach
1.  **Modify `extract_live_trades.py`**:
    *   Update `collect_live_trades` function.
    *   Change the key generation logic from `trade_id` to `f"{data.get('script_name')}_{trade_id}"`.
    *   This ensures uniqueness across different strategies.
2.  **Regenerate Data**:
    *   Run `extract_live_trades.py` manually to refresh `_live_trades.json`.
3.  **Verify**:
    *   Check `_live_trades.json` for the previously missing trade (ID "32" from `breakout_R_Rev_3...`).
4.  **Update Version**:
    *   Update `constants.py` to `V20260203_1000`.
