# Plan - Strategy Drill-Down from Activation Explorer

This plan describes how to link activations directly to their corresponding trades on the Live Orders page.

## 1. Understanding of Requirements
- **Activation Explorer**: Add a button/link to each strategy card that navigates to the Live Orders page.
- **Live Orders**: Filter the displayed trades by the strategy selected from the Activation Explorer.
- **Usability**: Add a visible search/filter bar on the Live Orders page so users can see and clear the filter.
- **Versioning**: Update to `V20260108_0130`.

## 2. Plan of Approach

### Live Orders (`TradeApps\breakout\live_trades.html`)
1.  **UI**:
    - Add a new control group in the header: `<label>Strategy Filter</label> <input type="text" id="strategyFilter" ...>`.
2.  **Logic**:
    - On page load, check for a `strategy` query parameter in `window.location.search`.
    - If found, populate the `strategyFilter` input.
    - Update `loadLiveTrades` to include `&strategy=${encodeURIComponent(filterVal)}` in the API fetch calls.
    - Trigger `loadLiveTrades` when the filter input is changed.

### Activation Explorer (`TradeApps\breakout\activations_explorer.html`)
1.  **UI**:
    - Add a "View Trades" button (or icon) to each strategy card in `renderCards`.
2.  **Logic**:
    - Implement `viewTrades(strategy)`:
        ```javascript
        function viewTrades(strategy) {
            window.location.href = `live_trades.html?strategy=${encodeURIComponent(strategy)}`;
        }
        ```

### Versioning
1.  Update `constants.py` and footer of all modified HTML files to `V20260108_0130`.
