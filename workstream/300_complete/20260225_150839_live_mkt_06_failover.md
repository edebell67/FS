## Description

## Objective
Ensure continuous data availability by automatically switching to backup providers on failure.

## Sub-tasks
- [ ] Create `src/core/failover.py` with:
  ```python
  class FailoverManager:
      def __init__(self, providers: List[BaseProvider])
      async def fetch_with_failover(self, symbol: str) -> MarketTick
      def get_primary_provider(self, asset_class: AssetClass) -> BaseProvider
      def mark_provider_failed(self, provider: str, reason: str)
      def mark_provider_healthy(self, provider: str)
  ```
- [ ] Implement failover triggers:
  - API error (4xx, 5xx)
  - Timeout exceeded (configurable threshold)
  - Data validation failure
  - Connection lost
- [ ] Add provider health scoring:
  - Track success/failure ratio
  - Track average latency
  - Auto-demote unhealthy providers
- [ ] Implement circuit breaker pattern:
  - Open after N consecutive failures
  - Half-open after cooldown period
  - Close on success
- [ ] Add automatic recovery checks
- [ ] Log all failover events
