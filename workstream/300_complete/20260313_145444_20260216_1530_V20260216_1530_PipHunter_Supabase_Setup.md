## Overview

This document details the complete setup process for integrating Supabase as the backend database for PipHunter. The setup will sync trading intelligence from JSON flat files to Supabase tables.

---

## Data Flow Architecture

```
SOURCE (Local)                  SUPABASE (Cloud)              CONSUMERS
──────────────                  ────────────────              ─────────
breakout/fs/json/live/
├── _top20.json ──────────┐
├── _targeted_strategies ─┼───► sync_to_supabase.py ───► Supabase Tables ───► PipHunter App
├── _live_trades.json ────┤                              ├── signals          ► Landing Page
├── _top_one.json ────────┘                              ├── market_bias      ► X/YouTube
                                                         ├── live_trades
                                                         ├── daily_summary
                                                         └── users (auth)
```

## Step 1: Create Supabase Project

### 1.1 Sign Up
1. Navigate to: https://supabase.com
2. Click "Start your project" or "Sign Up"
3. Sign in with GitHub (recommended) or email

### 1.2 Create New Project
1. Click "New Project"
2. Fill in:
   - **Organization**: Select or create one
   - **Project name**: `piphunter`
   - **Database password**: Generate a strong password and **SAVE IT**
   - **Region**: Select closest to your location (e.g., `eu-west-2` for UK)
3. Click "Create new project"
4. Wait 2-3 minutes for provisioning

## Step 2: Create Database Tables

### 2.1 Access SQL Editor
1. In Supabase dashboard, click **SQL Editor** (left sidebar)
2. Click "New query"

### 2.2 Run Schema Creation SQL

Copy and paste the following SQL, then click "Run":

```sql
-- ============================================================
-- PipHunter Database Schema
-- Version: V20260216_1530
-- ============================================================

-- ────────────────────────────────────────────────────────────
-- Table: signals
-- Purpose: Top performing strategies (synced from _top20.json)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS signals (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Core signal data
    pair VARCHAR(20) NOT NULL,              -- e.g., "GBP/AUD"
    product VARCHAR(50) NOT NULL,           -- e.g., "GBPAUD_C"
    strategy VARCHAR(100) NOT NULL,         -- e.g., "breakout_R_Rev_4_tp10.0_sl30.0"
    direction VARCHAR(4) NOT NULL,          -- "BUY" or "SELL"

    -- Performance metrics
    confidence INTEGER DEFAULT 0,           -- Win rate percentage (0-100)
    net_pnl DECIMAL(12,2) DEFAULT 0,        -- Total P&L
    buy_net DECIMAL(12,2) DEFAULT 0,        -- BUY side P&L
    sell_net DECIMAL(12,2) DEFAULT 0,       -- SELL side P&L
    buy_count INTEGER DEFAULT 0,            -- BUY trade count
    sell_count INTEGER DEFAULT 0,           -- SELL trade count
    buy_percent DECIMAL(5,2) DEFAULT 0,     -- BUY win rate
    sell_percent DECIMAL(5,2) DEFAULT 0,    -- SELL win rate
    trade_count INTEGER DEFAULT 0,          -- Total trades

    -- Metadata
    source_date DATE NOT NULL,              -- Date of data
    rank INTEGER,                           -- Position in top 20
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Indexes for fast queries
CREATE INDEX IF NOT EXISTS idx_signals_date ON signals(source_date DESC);
CREATE INDEX IF NOT EXISTS idx_signals_pnl ON signals(net_pnl DESC);
CREATE INDEX IF NOT EXISTS idx_signals_pair ON signals(pair);
CREATE INDEX IF NOT EXISTS idx_signals_direction ON signals(direction);

-- ────────────────────────────────────────────────────────────
-- Table: market_bias
-- Purpose: Market direction/sentiment (synced from _targeted_strategies.json)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS market_bias (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Bias data
    bias VARCHAR(10) NOT NULL,              -- "BUY", "SELL", "NEUTRAL"
    status VARCHAR(20),                     -- "STRONG", "MIXED"
    market_condition VARCHAR(50),           -- "CHOPPY", "TRENDING"
    day_bias VARCHAR(10),                   -- Today's overall bias
    recent_bias VARCHAR(10),                -- Recent trend direction
    ui_bias VARCHAR(10),                    -- UI display bias

    -- P&L context
    recent_buy_pnl DECIMAL(12,2),
    recent_sell_pnl DECIMAL(12,2),
    recent_buy_count INTEGER,
    recent_sell_count INTEGER,
    eligible_count INTEGER,

    -- Metadata
    source_date DATE NOT NULL,
    recorded_at TIMESTAMP DEFAULT NOW()
);

-- Index for time-based queries
CREATE INDEX IF NOT EXISTS idx_bias_date ON market_bias(source_date DESC);
CREATE INDEX IF NOT EXISTS idx_bias_time ON market_bias(recorded_at DESC);

-- ────────────────────────────────────────────────────────────
-- Table: live_trades
-- Purpose: Currently open trades (synced from _live_trades.json)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS live_trades (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Trade identification
    trade_id VARCHAR(50) NOT NULL,
    product VARCHAR(50) NOT NULL,
    direction VARCHAR(10) NOT NULL,         -- "LONG" or "SHORT"

    -- Strategy info
    script_name VARCHAR(100),
    source_strategy VARCHAR(100),
    source_screen VARCHAR(50),              -- "breakout", "grid_live"

    -- Price data
    entry_time TIMESTAMP,
    entry_price DECIMAL(18,8),
    current_price DECIMAL(18,8),

    -- P&L
    gross_pnl_pips DECIMAL(12,4),
    net_return DECIMAL(12,2),
    alt_net DECIMAL(12,2),

    -- Status
    status VARCHAR(20) NOT NULL,            -- "OPEN", "CLOSED"
    is_live_trade BOOLEAN DEFAULT FALSE,
    is_live_scheduled BOOLEAN DEFAULT FALSE,

    -- Parameters
    window_size INTEGER,
    pip_buffer DECIMAL(10,6),
    tp_pips DECIMAL(10,2),
    sl_pips DECIMAL(10,2),

    -- Blocking info
    trade_block_reason TEXT,

    -- Metadata
    source_file VARCHAR(200),
    source_date DATE NOT NULL,
    synced_at TIMESTAMP DEFAULT NOW()
);

-- Indexes
CREATE INDEX IF NOT EXISTS idx_live_trades_date ON live_trades(source_date DESC);
CREATE INDEX IF NOT EXISTS idx_live_trades_status ON live_trades(status);
CREATE INDEX IF NOT EXISTS idx_live_trades_product ON live_trades(product);

-- ────────────────────────────────────────────────────────────
-- Table: top_picks
-- Purpose: Best BUY/SELL signals (synced from _top_one.json)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS top_picks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    direction VARCHAR(4) NOT NULL,          -- "BUY" or "SELL"
    product VARCHAR(50) NOT NULL,
    strategy VARCHAR(150),
    pnl DECIMAL(12,2),

    source_date DATE NOT NULL,
    recorded_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_top_picks_date ON top_picks(source_date DESC);

-- ────────────────────────────────────────────────────────────
-- Table: daily_summary
-- Purpose: Daily aggregated stats for analytics
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS daily_summary (
    date DATE PRIMARY KEY,

    -- Aggregates
    signal_count INTEGER DEFAULT 0,
    total_pnl DECIMAL(12,2) DEFAULT 0,
    avg_confidence DECIMAL(5,2) DEFAULT 0,
    top_pair VARCHAR(50),
    top_strategy VARCHAR(100),

    -- Bias summary
    dominant_bias VARCHAR(10),
    bias_changes INTEGER DEFAULT 0,

    -- Trade stats
    live_trade_count INTEGER DEFAULT 0,
    total_trades_executed INTEGER DEFAULT 0,

    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- ────────────────────────────────────────────────────────────
-- Table: users (for PipHunter app authentication)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    -- Auth
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255),
    auth_provider VARCHAR(20) DEFAULT 'email',  -- "email", "google", "apple"
    email_verified BOOLEAN DEFAULT FALSE,

    -- Profile
    display_name VARCHAR(100),
    avatar_url VARCHAR(500),
    timezone VARCHAR(50) DEFAULT 'UTC',

    -- Subscription
    is_premium BOOLEAN DEFAULT FALSE,
    premium_expires_at TIMESTAMP,

    -- Timestamps
    created_at TIMESTAMP DEFAULT NOW(),
    last_login_at TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_users_email ON users(email);

-- ────────────────────────────────────────────────────────────
-- Table: user_streaks (gamification)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS user_streaks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,

    streak_type VARCHAR(50) NOT NULL,       -- "login", "signal_check"
    current_count INTEGER DEFAULT 0,
    best_count INTEGER DEFAULT 0,
    last_activity_date DATE,

    created_at TIMESTAMP DEFAULT NOW(),

    UNIQUE(user_id, streak_type)
);

-- ────────────────────────────────────────────────────────────
-- Table: user_xp (gamification)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS user_xp (
    user_id UUID PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE,

    total_xp INTEGER DEFAULT 0,
    current_level INTEGER DEFAULT 1,

    updated_at TIMESTAMP DEFAULT NOW()
);

-- ────────────────────────────────────────────────────────────
-- Table: user_challenges (gamification)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS user_challenges (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,

    challenge_id VARCHAR(50) NOT NULL,
    challenge_date DATE NOT NULL,
    progress INTEGER DEFAULT 0,
    completed BOOLEAN DEFAULT FALSE,
    claimed BOOLEAN DEFAULT FALSE,

    created_at TIMESTAMP DEFAULT NOW(),

    UNIQUE(user_id, challenge_id, challenge_date)
);

CREATE INDEX IF NOT EXISTS idx_user_challenges_date ON user_challenges(challenge_date DESC);

-- ────────────────────────────────────────────────────────────
-- Table: device_tokens (push notifications)
-- ────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS device_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,

    token VARCHAR(500) NOT NULL,
    platform VARCHAR(20) NOT NULL,          -- "ios", "android", "web"

    created_at TIMESTAMP DEFAULT NOW(),

    UNIQUE(user_id, token)
);

-- ────────────────────────────────────────────────────────────
-- Row Level Security (RLS) Policies
-- ────────────────────────────────────────────────────────────

-- Enable RLS on all tables
ALTER TABLE signals ENABLE ROW LEVEL SECURITY;
ALTER TABLE market_bias ENABLE ROW LEVEL SECURITY;
ALTER TABLE live_trades ENABLE ROW LEVEL SECURITY;
ALTER TABLE top_picks ENABLE ROW LEVEL SECURITY;
ALTER TABLE daily_summary ENABLE ROW LEVEL SECURITY;
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_streaks ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_xp ENABLE ROW LEVEL SECURITY;
ALTER TABLE user_challenges ENABLE ROW LEVEL SECURITY;
ALTER TABLE device_tokens ENABLE ROW LEVEL SECURITY;

-- Public read access for signal data (anyone can read)
CREATE POLICY "Public read signals" ON signals FOR SELECT USING (true);
CREATE POLICY "Public read bias" ON market_bias FOR SELECT USING (true);
CREATE POLICY "Public read live_trades" ON live_trades FOR SELECT USING (true);
CREATE POLICY "Public read top_picks" ON top_picks FOR SELECT USING (true);
CREATE POLICY "Public read daily_summary" ON daily_summary FOR SELECT USING (true);

-- Service role can insert/update signal data
CREATE POLICY "Service insert signals" ON signals FOR INSERT WITH CHECK (true);
CREATE POLICY "Service update signals" ON signals FOR UPDATE USING (true);
CREATE POLICY "Service delete signals" ON signals FOR DELETE USING (true);

CREATE POLICY "Service insert bias" ON market_bias FOR INSERT WITH CHECK (true);
CREATE POLICY "Service insert live_trades" ON live_trades FOR INSERT WITH CHECK (true);
CREATE POLICY "Service delete live_trades" ON live_trades FOR DELETE USING (true);
CREATE POLICY "Service insert top_picks" ON top_picks FOR INSERT WITH CHECK (true);
CREATE POLICY "Service delete top_picks" ON top_picks FOR DELETE USING (true);
CREATE POLICY "Service upsert daily_summary" ON daily_summary FOR ALL USING (true);

-- Users can only access their own data
CREATE POLICY "Users read own profile" ON users FOR SELECT USING (auth.uid() = id);
CREATE POLICY "Users update own profile" ON users FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Users read own streaks" ON user_streaks FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users manage own streaks" ON user_streaks FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users read own xp" ON user_xp FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users manage own xp" ON user_xp FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users read own challenges" ON user_challenges FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users manage own challenges" ON user_challenges FOR ALL USING (auth.uid() = user_id);

CREATE POLICY "Users manage own devices" ON device_tokens FOR ALL USING (auth.uid() = user_id);

-- ────────────────────────────────────────────────────────────
-- Verification Query
-- ────────────────────────────────────────────────────────────
-- Run this to verify tables were created:
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

## Troubleshooting

### Error: "Invalid API key"
- Check `.env` file has correct `SUPABASE_KEY`
- Ensure no extra spaces or quotes around the key

### Error: "relation does not exist"
- Run the SQL schema creation again
- Check you're connected to the correct project

### Error: "RLS policy violation"
- Use service_role key for sync script (not anon key)
- Check RLS policies allow insert/update

### Sync runs but no data appears
- Check `source_date` matches today's date
- Verify JSON files exist in `breakout/fs/json/live/{date}/`
