# Periodic Market Update Generator Plan (V20260224_1340)

## 1. Plan of Approach
The objective is to implement a periodic market update generator that uses the narrative rules defined in `.agent/skills/strategy-boxing-battle-pulse/SKILL.md` and `strategy-battle-punchy-updates` to surface punchy, live battle-style commentary to the "PERIODIC MARKET UPDATE" panel in the Breakout dashboard.

### Steps:
1. **Create Python Generator Script (`market_update_generator.py`)**:
   - This script will run periodically (e.g., via the existing `run_all_live_strategies.py` or a standalone loop) or an endpoint that can be triggered.
   - It will fetch the latest trade data for the current date and compute:
     - The overall BUY vs SELL net P&L (`totals`).
     - The last 30 minutes and previous 30 minutes BUY vs SELL net P&L (`windows`).
     - The market imbalance.
     - The strongest and weakest products.
     - Open trade pressure.
   - It will format a narrative string using the "Boxing Battle Pulse" style: "Bell: BUY X vs SELL Y. BUY presses." etc.
   - It will export this data into `fs/json/<mode>/<date>/_market_update.json` and append to `_market_update_history.json`.

2. **JSON Output Structure Optimization**:
   Ensure the output matches what the frontend currently expects:
   ```json
   {
       "exists": true,
       "update": {
           "headline": "LIVE Battle Pulse | Bias BUY | 14:30:00",
           "created_at": "...",
           "bias": "BUY",
           "market_condition": "TRENDING",
           "totals": {"buy_net": 120, "sell_net": -50, "imbalance": 170},
           "windows": {
               "last_30m": {"buy_net": 40, "sell_net": 10},
               "prev_30m": {"buy_net": 20, "sell_net": -5}
           },
           "narrative": "1. Bell: BUY... 2. BUY presses... Product impact: GBPEUR_C +80.0",
           "forecast": "BUY favored into next round (HIGH)."
       }
   }
   ```

3. **Background Execution**:
   - Integrate script into the regular cron/loop or add a small async thread in `trade_viewer_api.py` to generate the file every 5 minutes if it's stale.
