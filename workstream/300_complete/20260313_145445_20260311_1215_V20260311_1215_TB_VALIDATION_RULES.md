# Plan - Trade Bucket Validation Rules Enhancement (Backend)

[V20260311_1215]

## 1. Understanding of Requirements
The user wants to enforce strict validation rules for Trade Buckets (TB) in the backend (`trade_viewer_api.py`), matching the rules implemented in the frontend:
- **No single-row buckets allowed** under any circumstance.
- Single strategies must be represented by multiple metrics (e.g., [B] and [S]) to satisfy the minimum 2-row requirement.
- Multiple strategies can have any metric, but the total count must be >= 2.
- Avoid duplicate Strategy+Metric combinations in a bucket.

## 2. Plan of Approach
1.  **Modify `_series_value_at_or_before`**: Add a `field` parameter to allow retrieving `buy_net` and `sell_net` from historical series data.
2.  **Update `_run_tb_workflow_once`**:
    - If the family has only 1 strategy, split it into two rows: one for Buy Net and one for Sell Net.
    - Ensure each strategy entry includes a `metric` field.
    - Add a final check to ensure `len(processed_strategies) >= 2` before saving.
3.  **Update `_run_profile_match_workflow_once`**:
    - If `len(c_family) == 1`, automatically split into Buy and Sell rows regardless of requested metric (to satisfy the "no single row" rule).
    - If `metric == "buy_sell"`, split every family member into Buy and Sell rows.
    - Ensure `len(processed_strategies) >= 2` before saving.
4.  **Update `_run_top_x_multi_chart_workflow`**:
    - Implement the same splitting logic for `buy_sell` and single-strategy families.
    - Ensure `len(processed_strategies) >= 2` before saving.
5.  **Update `create_trade_bucket` API**:
    - Add a final validation check to reject any request that would result in a single-row bucket.
6.  **Update Version Number**: Increment version in `Constants.py`.
