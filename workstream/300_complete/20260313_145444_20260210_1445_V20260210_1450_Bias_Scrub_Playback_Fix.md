## 1. Understanding of Requirements

*   **ReferenceError**: The user encounters `resetSummaryPlayback is not defined` when interacting with the timeline playback controls. This likely stems from a script execution failure or improper function hoisting/scoping in the large `strategy_performance.html` file.
*   **Bias Discrepancy**: The bias indicator in the header (Global Bias) may differ from what the Performance Table suggests (Local Dominance) or what the user sees during playback. The user wants these to be synchronized and insightful.
*   **Aesthetics**: Maintain the high-fidelity Indigo/Amber glassmorphism look.

## 2. Plan of Approach

### Part A: Fix Playback Reference Errors
1.  **Function Migration**: Move all Summary Playback Engine functions (`resetSummaryPlayback`, `toggleSummaryPlayback`, etc.) to the absolute start of the `<script>` block in `strategy_performance.html`. This ensures they are hoisted and available immediately, regardless of potential runtime errors in later sections of the script.
2.  **State Initialization**: Group all global playback variables (`summaryFilterTime`, `isSummaryPlaying`, etc.) at the top as well.
3.  **Error Isolation**: Use try-catch blocks in critical initialization code to prevent a single failure from nuking the entire script execution.

### Part B: Synchronize Bias Monitor & Add Playback Scrubbing
1.  **Bias Scrubbing**: 
    *   Fetch `bias_history.json` when the Performance Summary modal is opened.
    *   In `projectSummaryData()` (which runs during timeline scrubbing), find the historical bias entry closest to the current playback time.
    *   Update a local "Timeline Bias" indicator in the modal to show the balance of power at that specific moment in history.
2.  **Bias Logic Alignment**: Ensure the UI's dominance indicators (green/red column highlights) align with the global `day_avg_net` logic used by the AI Picker, rather than recalculating it locally in a way that might conflict.
3.  **Picker Persistence**: Ensure the `automated_strategy_picker.py` is running to keep `_targeted_strategies.json` up to date.
