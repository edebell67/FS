# Plan - Use _top_one.json for Auto Activation (V20260102_2200)

This plan outlines the changes required to shift the backend auto-activation logic to use the summary leader file instead of manual JSON scanning.

## 1. Understanding of Requirements
- The current auto-activation logic scans all virtual trades to determine the top performer.
- A `_top_one.json` file already exists which identifies the current Buy and Sell leaders.
- The user wants the activation logic to consume this file directly to determine the "Top 1" (or Top N) strategy to activate.

## 2. Plan of Approach
- Locate `_perform_auto_activation_check` in `common.py`.
- Define the dynamic path for `_top_one.json` using the existing `json_base_dir` and date resolution.
- Load the leader file.
- Collate `top_buy` and `top_sell` into a candidate list.
- Format the keys correctly for `activations.json` (e.g., appending `_net`).
- Apply the `threshold_y` (Profitability Guard) to the leaders.
- If multiple candidates exist and `top_n_strategies` is 1, pick the single highest P&L performer.
- Update the system version.

## 3. List of Changes
- **`common.py`**:
    - Update `_perform_auto_activation_check` to load and use `_top_one.json`.
- **`constants.py`**:
    - Update `VERSION` to `V20260102_2200`.

## 4. Verification
- Verify that the activation system respects the `threshold_y` gate using the data from the leader file.
- Ensure the products are correctly assigned based on the leader file data.
