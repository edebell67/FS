# Plan: UI Restoration & Safety Exit Targets (V20260121_2015)

This plan fixes the "undefined" parameter issue in the UI and implements hardcoded safety exit targets ($200 TP, -$60 SL) as requested.

## 1. Understanding of Requirements
- The **Parm** column in the performance table shows `undefined`.
- The **Drill-down** filter (linking to individual trades) is broken due to key mismatches.
- Trades should **auto-close** if `net_return >= 200` or `net_return <= -60` by default.
- Reversal strategies (`_rev`) should close on opposite signals (to be handled in engine/SQL).

3.  **Modify `strategy_performance.html`**:
    *   Update mapping to use `buy_pct` and `sell_pct` from the API.
    *   Fix the `parm` mapping logic to prevent `undefined` display.

### Workstream 7: Hardened Exit Criteria (PostgreSQL)
1.  **Modify `db_scripts/pgsql_pnl_management.sql`**:
    *   Update `close_trades_target_reached()`:
        *   Add condition `(net_return >= 200)` for Take Profit (TP).
        *   Add condition `(net_return <= -60)` for Stop Loss (SL).
    *   Add "Reversal Logic" (Optional SQL side): If a newer trade enters for the same strategy/product, any existing trades in the opposite direction should be closed.
