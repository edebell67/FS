# Weighted Race Rank Filter Implementation

## 1. Understanding of Requirements
The user wants to filter the Leaderboard in `frequency_explorer.html` to show strategies that have "currently or previously hit a specified Rank" (1 to 5).
The default filter should be Rank 1.

To support this:
1.  **Backend**: `weighted_race.py` must track the **Best Rank** achieved by each strategy throughout the day's snapshots.
2.  **Frontend**: `frequency_explorer.html` must provide a UI control (Select 1-5) and filter the displayed table rows based on this `best_rank`.

## 2. Plan of Approach

### Backend (`weighted_race.py`)
*   Initialize a dictionary `best_rank` to track the minimum rank (1=best) for each strategy.
*   Update this tracking inside the main bucket loop.
*   Include `best_rank` in the final summary output.

### Frontend (`frequency_explorer.html`)
*   Add a `<select id="rankFilter">` to the Leaderboard panel filters.
*   Update `renderTable()` to filter items where `item.best_rank <= selectedRank`.
*   Implement a fallback for legacy data: if `best_rank` is missing, use `rank1_appearances > 0 ? 1 : 5`.
