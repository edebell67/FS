# Promotion Blocks Screen

## Goal
Add a dedicated diagnostics page to explain why trades were not promoted to L-trade, including the grid strategies in place at rejection time.

## Changes Applied

### 1) `TradeApps/breakout/fs/common.py`
- Added grid context snapshot capture at block time:
  - New helper: `_load_grid_live_entries()`
  - New helper: `_get_grid_live_context_snapshot(model, product)`
- On block, trade JSON now stores:
  - `trade_block_reason` (existing)
  - `trade_block_reason_detail` (new detailed diagnostics)
  - `grid_live_context_at_block` (new)

### 2) `TradeApps/breakout/fs/trade_viewer_api.py`
- Added endpoint:
  - `GET /api/promotion_blocks?mode=live|sim&date=YYYY-MM-DD&include_closed=true|false`
- Returns:
  - blocked trades list with reason/detail/context
  - current `grid_live` entries for selected mode

### 3) `TradeApps/breakout/fs/promotion_blocks.html` (new)
- New diagnostics page with matching app look-and-feel:
  - Uses `sidebar.css` + `pnl_graph.css`
  - Glass cards/panel/table style consistent with existing pages
- Features:
  - filters: mode/date/include_closed
  - summary counters
  - current grid context panel
  - blocked trades table with reason/detail
  - context modal (grid snapshot at block time)

### 4) `TradeApps/breakout/fs/sidebar.html`
- Added menu entry:
  - `Promotion Blocks` -> `promotion_blocks.html`

## Validation
- `python -m py_compile TradeApps/breakout/fs/common.py TradeApps/breakout/fs/trade_viewer_api.py`

## Notes
- The new endpoint is on the Flask trade viewer API service.
- If page/API is not immediately available, restart `trade_viewer_api.py` process serving UI pages.
