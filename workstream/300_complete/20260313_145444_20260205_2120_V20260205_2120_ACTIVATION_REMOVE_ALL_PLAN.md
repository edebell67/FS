# Activation Explorer: Remove All Functionality (2026-02-05)

This plan outlines the implementation of a "Remove All" button in the Activation Explorer UI to allow users to clear all trade activations for a specific run mode (LIVE or SIMULATION).

## 1. Understanding of Requirements
The user wants a global "Remove All" button on the Activation Explorer screen to facilitate bulk management and system resets. This button must be clearly visible, maintain the UI's aesthetic, and include security safeguards (confirmation dialogs).

## 2. Plan of Approach
1.  **Backend Integration**: Create a new API endpoint `/api/activations/remove_all` in `trade_viewer_api.py`. This endpoint will accept a `mode` parameter and clear all entries in the `activations.json` file for that specific mode.
2.  **UI Modification**: 
    *   Insert a "Remove All" button into the `controls-row` of `activations_explorer.html`.
    *   Apply a distinct "danger" style (red/pink accents) to distinguish it from the standard refresh action.
    *   Add a FontAwesome icon (`fa-trash-sweep`) for visual clarity.
3.  **Frontend Logic**: Implement an asynchronous JavaScript function `removeAllActivations()` that:
    *   Visualizes the current count of activations to be removed.
    *   Triggers a browser `confirm()` dialog to prevent accidental clicks.
    *   Calls the new backend API endpoint.
    *   Triggers a UI refresh upon success.
4.  **Versioning**: Update the system version in `Constants.py`.
