# Fix Virtual Trade Selection Logic for Single Leader Mode

## Goal Description

When `multi_vtrade` is set to `false`, the system currently hardcodes the selection to use the frequency-based API leader, ignoring the `vtrade_selection_type: "top_net"` configuration. This plan fixes the logic so that "Single Leader" mode correctly picks the top performer based on the configured selection type.

## Proposed Changes

### [common.py](file:///c:/Users/edebe/eds/TradeApps/breakout/common.py)

**Modify `_manage_virtual_trades` function:**
- In the `else` block (when `multi_vtrade` is `false`):
    - Check the `vtrade_selection_type`.
    - If `top_net`, compare `top_buy` and `top_sell`.
    - Select the one with the higher absolute PnL.
- Update Version in `Constants.py` to `V20260114_0500`.
