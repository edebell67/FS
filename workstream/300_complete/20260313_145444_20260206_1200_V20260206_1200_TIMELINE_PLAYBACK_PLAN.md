# Plan: Strategy Performance Timeline Playback Restoration [V20260206_1200]

The goal is to restore and enhance the timeline playback feature on the Strategy Performance page, allowing users to "replay" the day's performance summary, similar to the functionality in `multi_chart.html`.

## 1. Requirements

- Add/Enable a timeline bar (slider) on `strategy_performance.html`.
- Enable "Play", "Pause", "Reset", and speed controls.
- When playback is active, the stats table should update to reflect the state of all strategies at that specific point in time.
- source of truth for historical snapshots: `_summary_net.json`.

## 2. Plan of Approach

### Frontend (strategy_performance.html)
1.  **UI Restoration**:
    - Unhide `.time-playback-container`.
    - Refine styles to match the premium dark/glassmorphism theme.
2.  **Data Fetching**:
    - Implement `fetchSummaryNet()` to download the date-specific `_summary_net.json`.
    - This file contains the history of `net`, `buy_net`, `sell_net`, `buyPercent`, etc., for all strategies.
3.  **Playback Engine**:
    - Implement `updateStatsFromPlayback(targetTime)`:
        - For each strategy/product in the `summaryData`, find the snapshot where `t <= targetTime`.
        - Use the values from that snapshot to update the table row.
    - Link `timeSlider` and playback buttons to this engine.
4.  **Integration**:
    - Update `loadStats()` to fetch the summary data when a date is selected.
    - Ensure `initializeTimePlayback()` sets the min/max bounds based on actual data points.
    - Auto-disable live refresh when playback is active.
