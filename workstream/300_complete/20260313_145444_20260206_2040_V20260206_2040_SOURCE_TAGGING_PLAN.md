# Plan: Source Attribution for AI Picker Pushes [V20260206_2040]

The goal is to ensure that when a user activates a strategy after finding it on the AI Picker screen, the activation is tagged with `source: ai_picker` instead of the generic `source: ui`.

## 1. Requirements

- Clicking "View / Push" on `ai_picker.html` must pass a reference to the Performance page.
- `strategy_performance.html` must detect this reference.
- When the user toggles an "Auto" button on the Performance page, the `source` must be sent to the API.

## 2. Plan of Approach

1.  **Modify `TradeApps/breakout/fs/ai_picker.html`**:
    - Change button link to: `strategy_performance.html?search=${s.strategy}&from=ai_picker`.
2.  **Modify `TradeApps/breakout/fs/strategy_performance.html`**:
    - In `init()`, parse the URL to check for `from=ai_picker`.
    - Create a global `referredSource` variable.
    - Update `toggleAutoActivation` to use `referredSource || 'ui'` as the `source` value in the activation payload.
3.  **Update `TradeApps/breakout/fs/constants.py`**:
    - Increment version to `V20260206_2040`.

## 3. List of Changes

- **`TradeApps/breakout/fs/ai_picker.html`**:
