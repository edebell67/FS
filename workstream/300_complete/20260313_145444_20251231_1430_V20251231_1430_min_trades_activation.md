# Plan: Add Minimum Trade Count to Auto-Activation

This document outlines the plan to add a minimum trade count requirement for strategy auto-activation.

## 1. Understanding of Requirements

*   Currently, the backend auto-activates strategies based solely on P&L (`total_pnl > threshold_y`).
*   The requirement is to ensure a strategy has completed a minimum number of trades (sample size) before it is eligible for auto-activation.
*   This sample size should be configurable.

## 2. Plan of Approach

1.  **Modify `config.json`**:
    *   Add a new field `"min_auto_trades": 5` (defaulting to 5 for statistical significance).
2.  **Modify `common.py`**:
    *   In `_perform_auto_activation_check`:
        *   Retrieve `min_auto_trades` from configuration.
        *   Calculate the `total_count` (sum of counts across all products) for each strategy.
        *   Update the candidacy filters for both **Net** and **Alt** paths to require `total_count >= min_auto_trades`.
3.  **Update Version**:
    *   Update `VERSION` in `constants.py` to `V20251231_1430`.
