# 20251222_1400_V20251222_1400_Daily_Target_Limit

## 1. Understanding of Requirements
The user wants to implement a daily profit target. If the combined P&L of all of today's "L" (Live) trades exceeds a configurable `daily_target` amount, the system should stop generating any new Live trades for the rest of the day.

This requires:
1.  A new `daily_target` setting in `config.json`.
2.  A new function to calculate the total P&L of today's live trades.
3.  An integration of this check into the live order generation logic.

## 2. Plan of Approach
1.  **Modify `config.json`**:
    *   Add `daily_target` with a default value of `400`.
2.  **Modify `TradeApps/breakout/common.py`**:
    *   Create a new function `_get_todays_live_pnl(json_base_dir)`. This function will scan today's trade files, sum the `net_return` of all trades where `is_live_trade` is true, and return the total.
    *   Modify `_handle_live_orders`: Before generating any trade, it will call `_get_todays_live_pnl`. If the returned P&L is greater than or equal to the `daily_target` from the config, it will print a "Daily Target Reached" message and immediately return, preventing any new L trades.
3.  **UI Update (`trade_viewer.html`)**:
    *   Add the `daily_target` field to the configuration modal so it can be viewed and edited from the UI.
