# Plan: Snap-to-Leader Activation (V20260129_1042)

This plan implements a "Snap-to-Leader" behavior where clicking the LIVE button selects the current top performer on a card and locks it in.

## 1. Requirements
- When the LIVE button is clicked (transitioning to ON), identify the strategy on the card with the highest value for the selected metric.
- Write only that specific strategy to `grid_live.json`.
- The selection is "Sticky": it does not change until the LIVE button is clicked again (OFF/ON) or a manual change is made.

## 2. Plan of Approach

### UI Logic (`multi_chart.js`)
- [x] Modify `toggleLiveMode(event, groupName)`.
- [x] If entering LIVE mode:
    - Iterate through overlays in `groupName`.
    - Calculate current delta (End - Start) for each using `processedSeries`.
    - Identify the winner.
    - Pass this winner to `liveMonitor.activateGroup(groupName, winner)`.
- [x] If leaving LIVE mode:
    - Pass empty list to `liveMonitor.activateGroup(groupName, null)`.

### Live Monitor Logic (`LiveMonitor.js`)
- [x] Update `activateGroup(groupName, modelDef)` to store the specific selection.
- [x] The `update()` method will no longer re-evaluate leaders; it will only maintain the "Sticky" selection made at click-time.

### Versioning
