# Migration Plan: SQL Server to PostgreSQL Object Parity (V20260305_1800)

## Objective
Achieve 100% schema parity for the `tradedb` database between SQL Server (tcp:EDS,1433) and PostgreSQL (localhost:5432). This is required to facilitate full data migration and functional transition.

## Scope
- **Tables**: 57 missing tables.
- **Views**: 89 missing views.
- **Procedures**: 130 missing procedures.

## Technical Strategy
1. **Automated Schema Translation**: Use a Python orchestrator to read T-SQL `.sql` source files and apply regex-based mapping for PostgreSQL compatibility.
2. **Sequential Implementation**: 
    - Phase 1: Tables (Base objects).
    - Phase 2: Views (Analytical layer).
    - Phase 3: Procedures (Logic layer).
3. **Data Type Mapping**:
    - `DATETIME` -> `TIMESTAMPTZ`
    - `NVARCHAR(MAX)` -> `TEXT`
    - `BIT` -> `BOOLEAN`
    - `FLOAT` -> `DOUBLE PRECISION`
    - `UNIQUEIDENTIFIER` -> `UUID` (or `VARCHAR(50)` for legacy compatibility)
    - `DECIMAL` -> `NUMERIC`
