# Plan: Implementation 2.1 - FTS5 Search Indexing and Querying [V20260219_1700]

**Project**: bizPA (Voice-First Small Trader Capture App)  
**Objective**: Implement PostgreSQL Full-Text Search (FTS) for fast retrieval of items and create the `/api/v1/search` endpoint.  
**Version**: V20260219_1700

## Approach
1.  **Index**: Instead of SQLite's FTS5, use PostgreSQL's `tsvector` and `GIN` indexes on `extracted_text`, `raw_note`, and `label_name`.
2.  **Mapping**: Create a view `vw_item_search` that joins `capture_items` with their `labels` for easier indexing.
3.  **Search Logic**: Use `websearch_to_tsquery` to allow natural language search strings.
4.  **Endpoint**: `POST /api/v1/search` will accept a `query` string and optional filters (date, type).
5.  **Voice Integration**: The `voiceController` will route search intents (e.g., "Show unpaid invoices") to the search service.
