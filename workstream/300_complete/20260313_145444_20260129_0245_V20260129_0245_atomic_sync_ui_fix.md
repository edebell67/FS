# Plan: Atomic Trade Sync & UI Restoration (V20260129_0245)

## 1. Problem Statement
The user reported two critical issues:
1. **Over-trading/Sync Collision**: The strategy scripts were overwriting JSON files every tick, effectively deleting the `is_live_trade: true` flag set by the grid monitor or manual promotion. This led to "empty" live screens and redundant order attempts.
2. **Missing UI Markers**: The white dots representing trades were missing from the Multi-Chart Grid.
3. **Redundant Logic**: The previous "cooldown" solution was rejected as it didn't address the root cause: identity and state persistence.

## 2. Plan of Approach

### Phase 1: Robust Data Persistence (The "Atomic Merge")
Instead of a simple overwrite, `common.py` will now perform a **Reload-Merge-Save** operation.
- **Merge Logic**: When saving `_op.json`, the script will first read the existing file from disk. If fields like `is_live_trade`, `order_sent_net`, or `in_trade_entry_price` are present on disk (set by the monitor), they are **merged** into the strategy's memory and preserved in the final write.
- **Identity Protection**: This ensures that once a trade is "promoted" to live, it stays live regardless of strategy restarts or tick updates.

### Phase 2: UI Aesthetic Restoration
Modify `multi_chart.js` to match the "Global P&L" graph aesthetics.
- **Trade Dots**: Set a default `pointRadius: 3` for all points to show the trade history.
- **Live Markers**: Implement large white markers (`radius: 6`, `rectRounded`) for positions identified as live.
- **Universal Markers**: Enable marker fetching for all series on a card, not just the "leader".

### Phase 3: Cleanup & Safety
- **Remove Cooldown**: Revert all cooldown-related code as requested.
- **Process Safety**: Only restart background processes if the main monitor `verify_algo_execution_and_restart_02.py` is stable.
