# Plan: Add All Windows Function (V20260220_0036)

## 1. Understanding of Requirements
The user wants a new button/functionality on the multi-chart cards to add all strategies that share the same base strategy prefix (e.g., `breakout`, `breakout_R`) and parameters (e.g., `tp10.0_sl20.0`), but have different window sizes (e.g., 2, 3, 4...). This will quickly add variations like `breakout_3_tp10.0_sl20.0` and `breakout_4_tp10.0_sl20.0` when `breakout_2_tp10.0_sl20.0` is currently displayed in the card.

## 2. Plan of Approach
1.  **Create New Function (`addAllWindows`)**:
    Add `addAllWindows(groupName)` in `multi_chart.js`, `multi_chart_v2.js`, and `multi_chart_v3.js`. 
    *   This function extracts the base strategy prefix, the TP, and the SL from the active overlay in the card using regex `(/^([a-zA-Z_]+)(?:_R|_Rev|_R_Rev)?(?:_\d+|_Rev_\d+|_R_\d+|_R_Rev_\d+)?_tp([\d\.]+)_sl([\d\.]+)$/)`. Wait, it's simpler to just match the window parameter which is `_(\d+)_` before `tp`.
    Let's refine the regex: `^(.*?)_(\d+)_tp([\d\.]+)_sl([\d\.]+)$`. The prefix is group 1, window size is group 2, TP is group 3, SL is group 4. 
    Actually, maybe just replace the window number part in the string.
    *   Iterate through `processedSeries` to find all strategies matching the same prefix, TP, and SL. 
    *   Add matches missing from active overlays and call `updateCharts()`.
2.  **Add UI Button**:
    Add a new button inside `createGroupCard(groupName)` in all three JS files, calling `addAllWindows('${groupName}')`.
3.  **Update Constants**:
    Update `VERSION` in `fs/constants.py` and `db/constants.py` to `V20260220_0036`.
