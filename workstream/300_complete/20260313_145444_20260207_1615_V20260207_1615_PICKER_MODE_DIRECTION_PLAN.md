# Plan: AI Picker Run Mode & Directional Activation Fix [V20260207_1615]

The user noted three issues: 
1. AI Picker needs to respect the `run_mode` from `config.json`.
2. Short trades executed even though only "Auto Buy" was active.
3. Open trade timestamps are out by 1 hour.

## 1. Requirements

- `ai_picker.html` must fetch `config.json` via `/api/config` and adjust data paths accordingly.
- `common.py`'s `grid_live` activation must respect the `activations.json` directional toggles (Buy/Sell) if `bypass_criteria_check` is NOT "ultra".
- AI Picker display must correctly handle UTC vs Local time discrepancies.

## 2. Plan of Approach

### A. AI Picker Run Mode Support
1.  **Modify `TradeApps/breakout/fs/ai_picker.html`**:
    - Add a `fetchConfig()` function to get the current mode.
    - Update `updateData()` to use `json/${mode}/${dateStr}/_targeted_strategies.json`.
    - Add a mode indicator in the header (LIVE / SIM).

### B. Directional Activation for Grid Live
1.  **Modify `TradeApps/breakout/fs/common.py`**:
    - In `_trigger_grid_live_activation`, add a check for direction-specific activation. 
    - Even if a strategy is in `grid_live.json`, it should only auto-activate if the corresponding `_buy_net` or `_sell_net` flag is set in `activations.json`, UNLESS `bypass_criteria_check` is set to "ultra" (which implies full automation).
    - This ensures that if the user only highlights "Auto Buy" in the UI, "Auto Sell" trades from the grid stay in virtual mode.

### C. Time Offset Fix
1.  **Modify `TradeApps/breakout/fs/automated_strategy_picker.py`**:
    - Use `datetime.utcnow()` instead of `datetime.now()` for `scan_latest_time` to align with the UTC timestamps recorded in trade files.
    - This should resolve the 1-hour discrepancy seen by the user.

### D. Documentation
1.  **Update `TradeApps/breakout/fs/constants.py`**:
    - Increment version to `V20260207_1615`.
