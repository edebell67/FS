# Plan: Resize Trade Modal and Address Activation Logic

This document outlines the changes to improve the clarity of the "LIVE OPEN Trades" modal and provides an explanation for the auto-activation logic.

## 1. Understanding of Requirements

*   **Modal Resize**: The user wants the "LIVE OPEN Trades" modal (and likely the general trade modal) to be more readable/clear. The provided image suggests making it fill more of the screen to accommodate the wide data table.
*   **Activation Logic Clarification**:
    *   If `Net Activation Mode = auto` AND `Auto Net Check = true`:
        *   Will `activations.json` be updated?
        *   Will L-trades be generated?
        *   Will the `manual` flag be `false`?

## 2. Answers to Logic Questions

1.  **`activations.json` Update**: Yes. The backend (`_perform_auto_activation_check`) runs Every 60 seconds. It evaluates performance and updates `activations.json` automatically.
2.  **`manual: false`**: Yes. When the system auto-activates a strategy, it sets the `manual` flag to `false`. This distinguishes it from strategies you activated by clicking the buttons in the UI.
3.  **L-trade Generation**: Yes. Once the backend sets a strategy to "active" in the JSON file, the trading scripts detect this in their next loop. If a trade setup occurs while active, it will generate an L-trade order.
