# Profitability Guardrail for Order Generation

**Date:** 2025-12-17 09:10
**Version:** 64.4.3
**Description:** Add a "Profitability Guardrail" to the activation checkboxes. Orders should only be generated if the cumulative closed P&L for the specific strategy/direction/type is positive (> 0).

## 1. Understanding of Requirements
*   **Condition**: Generate TWS orders (`tradeable.json`) *only* if the cumulative P&L for the target category (Buy/Sell x Net/Alt) is strictly positive.
*   **Scope**: logic applies to 'Net' orders (checking Net P&L) and 'Alt' orders (checking Alt Net P&L) independently.
*   **Behavior**:
    *   First trade of session (P&L = 0) -> Blocked (Paper Trade only).
    *   Subsequent trades -> Allowed only if cumulative history is profitable.

## 2. Plan of Approach
1.  **Add Accumulators**:
    *   In `__init__`, add `self.cum_buy_net`, `self.cum_sell_net`, `self.cum_buy_alt`, `self.cum_sell_alt`.
2.  **Update on Close**:
    *   In `close_trade`, update usage of these accumulators based on trade direction.
3.  **Check on Entry**:
    *   In `check_and_enter`, prior to calling `_create_tradeable_json`:
        *   Determine context (Buy/Sell).
        *   Check if relevant accumulator > 0.
        *   Only then proceed with file generation and setting `order_sent` flags.

## 3. List of Changes
*   **`breakout.py`**: Init, Close, Entry updates.
*   **`breakout_R.py`**: Init, Close, Entry updates.
*   **`breakout_Rev.py`**: Init, Close, Entry updates.
*   **`breakout_R_Rev.py`**: Init, Close, Entry updates.
