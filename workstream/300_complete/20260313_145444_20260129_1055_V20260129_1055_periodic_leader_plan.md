# Plan: Continuous Periodic Leader Election (V20260129_1055)

This plan restores the "Leader Election" capability but ensures it is stable, controlled, and works with the flattened `grid_live.json` format.

## 1. Requirements
- If the LIVE button is active for a card, the system must continuously monitor the strategies on that card.
- Every X minutes (defaulting to 1 minute), it must re-evaluate which strategy is the "Leader" (top performer).
- If the leader changes, it must update `grid_live.json` with the new leader for that card.
- Use the flattened list format in `grid_live.json`.
