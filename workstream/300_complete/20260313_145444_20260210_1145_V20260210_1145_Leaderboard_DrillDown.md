[V20260210_1145] Leaderboard Drill-Down Fix
**Current Status**: COMPLETED by Claude Code (2026-02-10 11:45)
**Objective**: Enable drill-down functionality from the "Best Parameters" leaderboard, allowing users to view all trades associated with a specific parameter set, regardless of product or strategy type.

## 1. Issue Analysis
The user reported "cant drill down from attached image" regarding the leaderboard. The leaderboard items were purely informational and lacked interactive `onclick` handlers. Additionally, the existing `showDrillDown` function required specific product and strategy names, which didn't align with the aggregated nature of the leaderboard parameters (which span multiple strategies).

## 2. Solution Implemented
1.  **Updated `showDrillDown`**: Modified the filtering logic to accept `'all'` as a wildcard for both `product` and `appName`. This allows the modal to display trades filtered only by the parameter string.
2.  **Raw Parameter Capture**: Updated the `paramMap` aggregation in `renderSummaryGrid` to capture the original raw parameter string (`parm_raw`) alongside the statistical data.
3.  **Clickable Leaderboard Items**: Updated the Leaderboard HTML generation to:
    *   Add `style="cursor: pointer;"` and a tooltip title.
    *   Add an `onclick` handler that calls `showDrillDown('all', 'all', rawParam, direction)`.
    *   Dynamically set the direction filter based on the current leaderboard metric (e.g., if viewing "Sell" leaderboard, drill-down shows Sell trades).

## 3. List of Changes
*   **`TradeApps/breakout/fs/strategy_performance.html`**:
    *   [x] Modified `showDrillDown` to support strict filtering with 'all' wildcards.
    *   [x] Updated `renderSummaryGrid` aggregation to store `raw` parameter string.
    *   [x] Updated Leaderboard HTML loop to include `onclick` events invoking `showDrillDown`.
