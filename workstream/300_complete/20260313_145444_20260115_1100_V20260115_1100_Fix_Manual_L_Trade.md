# Implement Manual Override for L-Trade Generation

## Goal Description
The "Manual Auto Buy" feature in the UI is intended to force a live trade even if system constraints (like `max_live_trades` or profitability guards) would normally block it. Currently, the backend ignores the "Manual" flag from `activations.json`, causing manual overrides to fail silently if limits are reached.
This change will ensure that trades marked as "Manual" in `activations.json` (set via the UI) bypass the `max_live_trades` check and Profitability Guard.

## User Review Required
> [!IMPORTANT]
> This change allows "Manual" trades to exceed the `max_live_trades` limit configured in `config.json`. Ensure this is the desired behavior for manual interventions.

> [!NOTE]
> The existing `bypass_criteria_check` setting in `config.json` controls whether strategy entry conditions (e.g. price thresholds) are ignored. It **does not** currently bypass trade limits like `max_live_trades` or profitability guards. This plan specifically addresses bypassing those system-level limits for manual trades.

## Proposed Changes

### TradeApps/breakout

#### [MODIFY] [common.py](file:///c:/Users/edebe/eds/TradeApps/breakout/common.py)
1.  **Add `_get_activation_details`**: Create a helper method in `BaseBreakoutStrategy` to retrieve the full activation details (active status, manual flag, etc.) to avoid repeated lookups.
2.  **Update `_is_active`**: Refactor to use `_get_activation_details`.
3.  **Update `_handle_live_orders`**:
    -   Retrieve activation details including the `manual` flag.
    -   If `manual` is True:
        -   Bypass the `_is_active` check (it's implied active).
        -   Bypass the `_is_profitability_guard_passed` check.
        -   Pass `manual_override=True` to `_create_tradeable_json`.
    -   If `manual` is False, proceed with normal checks.
4.  **Update `_create_tradeable_json`**: Add `manual_override` parameter (default `False`) and pass it to `_create_l_trade_order` as `bypass_max_trades`.
5.  **Update `_create_l_trade_order`**:
    -   Add `bypass_max_trades` parameter (default `False`).
    -   SKIP the `max_live_trades` check if `bypass_max_trades` is True.
    -   Add logging to indicate a manual override trade was created.
