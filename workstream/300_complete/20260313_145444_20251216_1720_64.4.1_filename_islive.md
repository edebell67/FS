# Update Filename Format and Add is_live_trade Field

**Date:** 2025-12-16 17:20
**Version:** 64.4.1 (Estimate)
**Description:** Update the naming convention for generated order JSON files sent to TWS/trades_rt3, and flag activated trades in the local JSON logs.

## 1. Understanding of Requirements

1.  **Order Filename Format**: The JSON files generated for trade execution (sent to `trades_rt3`) must follow the format: `'breakout'_{strategy_name}_{strategy parameters}_{signal/direction}_{timestamp}_tradeable.json`.
    *   Parameters format: `window_buffer_tp_sl` (e.g. `5_0.0001_15.0_6.0`).
    *   Timestamp format: `yyyymmddHHMM`.
2.  **Local Trade Log Metadata**: The local JSON files (stored in `json/live/...`) must include a new boolean field `"is_live_trade": true` (or false) to indicate if the trade triggered an order generation (i.e., was "activated").

## 2. Plan of Approach

1.  **Modify `_create_tradeable_json` in all strategy scripts**:
    *   Reconstruct the filename string to match the requested format.
    *   Use `datetime.now().strftime('%Y%m%d%H%M')` for the timestamp.
    *   Ensure the `action` (Buy/Sell) reflects the flipped logic if applicable.
2.  **Modify `_save_trade_json` in all strategy scripts**:
    *   Add `"is_live_trade"` key to the `trade_data` dictionary.
    *   Value derived from `open_trade['order_sent_net']` OR `open_trade['order_sent_alt']`.
