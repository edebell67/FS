# Fix Summary: Auto Buy "A" Not Executing Trades

## Problem Identified

**Root Cause**: Strategy name mismatch between `grid_live.json` and actual running strategies.

### What Was Happening:
1. ✅ UI "Auto Buy" button correctly updated `grid_live.json`
2. ✅ Strategy scripts were running 24/7 and creating trades
3. ❌ **Trades were NOT being promoted to live** because:
   - `grid_live.json` had: `"model": "breakout_Rev_2_tp10.0_sl20.0"`
   - Actual trades had: `"script_name": "breakout_2_tp10.0_sl20.0"` (no "Rev")
   - The bypass logic in `common.py` (line 687-703) checks if the strategy exists in `grid_live.json`
   - Lookup failed → Trade blocked with reason: `"Bypass blocked (strategy not monitored in grid_live)"`

## Solution Applied

**Added missing strategy entries to `grid_live.json`**:

Added 4 entries for `breakout_2_tp10.0_sl20.0` (the actual running strategy):
- EURAUD_C
- GBPEUR_C  
- EURNZD_C
- GBPAUD_C

All with `"metric": "buy_net"` to match your Auto Buy activation.

## What Happens Next

Within the next 60 seconds (next poll cycle):

1. **Strategy scripts** will reload `grid_live.json` and detect the new entries
2. **Existing open trades** will be checked via `_check_grid_live_toggle()` (line 769)
3. **L-Trade orders** will be created via `_create_l_trade_order()` (line 2255)
4. **max_live_trades guard** will enforce the limit of 1 concurrent live trade
5. **First qualifying trade** will get promoted, others will be blocked with:
   ```
   [GUARD] Max live trades (1) reached. Currently have 1 open. Skipping...
   ```

## Key Takeaway

The "Auto Buy = A" button works correctly, but you need to ensure:
1. **Strategy names in grid_live.json EXACTLY match** the `script_name` in trade files
2. **All 4 strategy scripts are running** (breakout, breakout_Rev, breakout_R, breakout_R_Rev)
3. **max_live_trades limit is respected** (currently set to 1)

The UI shows "A" for strategies in grid_live.json, but if the actual running strategy has a different name, trades won't be promoted.
