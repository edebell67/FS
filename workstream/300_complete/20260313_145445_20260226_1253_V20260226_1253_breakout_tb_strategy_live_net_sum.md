# Task: Add Live Net Sum for L-Trades per Strategy in Trade Bucket UI

## 1. Understanding of Requirements
The user wants the "Live Net" column in the Trade Bucket UI strategy list to dynamically display the sum of L-Trades' net returns for each specific strategy. Currently, it defaults to `--` if the backend `live_trade_net` property is missing or 0. The sum must reflect exactly the L-trades valid for that strategy within that bucket, evaluating to `0.00` if no L-trades exist.

## 2. Plan of Approach
1. **Analyze `trade_bucket.html` (`renderBuckets`)**: 
   - A list of valid L-trades for the entire bucket (`bucketLTrades`) is already constructed just before the strategies `.map()` loop.
   - For each strategy `s` in the loop, we will parse its unique key using `parseBucketKey()`.
   - We will filter `bucketLTrades` to find the subset of trades matching this parsed key using the existing `tradeMatchesBucketStrategy()` function.
   - We will sum the `net_return` property of this subset to determine the `stratLiveNet`.
2. **Update UI Formatter**:
   - We will change the `liveNetDisplay` ternary operator. If we compute the live net, we want to always display `0.00` when it evaluates as 0, rather than reverting to `--`.
3. **Update Versions**:
