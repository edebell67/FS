[V20260210_1130] Fix Level 4 Drill-Down
**Current Status**: COMPLETED by Claude Code (2026-02-10 11:30)
**Objective**: Fix the issue where "cannot see individual trades" when attempting to drill down from the Product level in the Performance Hierarchy.

## 1. Issue Analysis
The user reported that clicking on a product row in the hierarchy (Level 4) did not show the trades.
Upon inspection, the `onclick` handler `showDrillDown` was using unescaped string values for `prodKey`, `strategy`, and `parm_raw`. If any of these contained special characters (like single quotes), it would cause a JavaScript syntax error, preventing the modal from opening.

## 2. Solution Implemented
1.  **Escape Helper**: Added a helper function `esc(s)` to `renderSummaryGrid` that escapes single quotes (`' -> \'`).
2.  **Updated Onclick**: Modified the Level 4 row generation to wrap all string arguments in `esc()` within the `onclick` attribute.
    *   `showDrillDown('${esc(prodKey)}', '${esc(prodNode.meta.strategy)}', '${esc(prodNode.meta.parm_raw)}', 'all')`

## 3. List of Changes
*   **`TradeApps/breakout/fs/strategy_performance.html`**:
    *   [x] Added `esc` helper function.
    *   [x] Updated Level 4 row rendering loop to safe-guard `onclick` parameters.

## 4. Verification
