[V20260204_2120] Automatic Trade Bucket Promotion & Periodic Sync

## 1. Requirement
The user wants Trade Bucket strategies to show up in the `grid_live.json` (and thus the Dashboard grid) automatically when buckets are live. Currently, promotion only happens at the exact moment a bucket is toggled "LIVE". If the bucket is already live and its performance improves later to meet the threshold, it is never added to the grid.

Additionally, the user wants Frequency updates (Rank #1) to be suspended when Buckets are active, which is currently done via a config flag, but stale frequency entries are still lingering in the grid.

## 2. Plan of Approach

### Background Reconciliation
Instead of a complex background thread, I will implement a "Just-in-Time" reconciliation logic inside the `/api/grid_live` GET route. Since the frontend `LiveMonitor` polls this route every few seconds, this ensures the grid is updated whenever the user is looking at it.

### Reconciliation Logic (`_reconcile_active_buckets`):
1.  **Check Config**: Check if `rank_alert_suspended` is true OR if any trade buckets are live.
2.  **Load Buckets**: Load all trade buckets for the current trading date.
3.  **Find Live Buckets**: Filter for buckets where `live` is `True`.
4.  **Mutual Exclusion**:
    - If live buckets exist: Force remove all strategies from `grid_live.json` that have a `source` starting with `frequency_` or `rank_alert_`.
5.  **Evaluate Bucket Leaders**:
    - For each live bucket, use `_calculate_bucket_strat_perf` to find the current top performer since `start_time`.
    - If the leader meets `minimum_difference`:
        - check if it's already in the grid with the correct bucket source.
        - if not, add it.
6.  **Save & Sync**: If changes were made, save `grid_live.json` and trigger `_sync_grid_to_activations`.
