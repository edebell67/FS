# Plan: Rank #1 Alert System (Frequency Explorer)

## Objective
Implement a real-time alert system in `frequency_explorer.html` that notifies the user when a new product/strategy combination takes the #1 Rank position.

## Features
*   **Detection**: Automatically detect when the #1 leader changes during auto-refresh updates.
*   **UI**: Display a non-modal, bright yellow prompt on top of the application.
*   **Intervention**: Provide 5 distinct actions for the user to respond to the new leader.

## User Interface
*   **Style**: Top-center, fixed position. Background color `#FFEB3B` (Bright Yellow). Text color `black`. High z-index.
*   **Content**:
    *   Header: "New Rank #1 Detected!"
    *   Detail: "[Product] [Strategy]"
    *   Net P&L: "[+120.50]"
    *   Detail: "[Product] [Strategy]"
    *   Net P&L: "[+120.50]"
    *   Buttons: 5 Action buttons (Column or Row layout).
    *   **Checkbox**: "Repeat selected option in future" (Matches new requirement).

## 4. Auto-Action / Remember Choice
*   **Logic**: If the user checks the box, the selected action is saved to `config.json` as `rank_alert_default_action`.
*   **Future Alerts**:
    *   If `rank_alert_default_action` is present, the alert is **skipped** (or shown as a temporary toast) and the action is executed automatically.
    *   Exception: "No Action" with repeat might just mean "Disable alerts" or "Ignore silently".
    *   Exception: "No more updates" is effectively "Disable".
*   **Reset**: Need a way to clear this preference (e.g., in a settings modal or config file).

## Actions
1.  **No Action**: Simply close the alert.
2.  **Monitor #1 Only**:
    *   Action: **Replace** the entire `grid_live.json` monitoring list with ONLY this new item.
    *   Endpoint: `POST /api/grid_live` (Need to support "overwrite" mode or clear list first).
3.  **Monitor #1 Priority**:
    *   Action: Add the item to `grid_live.json` with a specific tag `priority: true` (or ensure it is at the top of the list).
    *   Implementation: Append to list, but maybe frontend logic highlights it.
4.  **Add to Monitor List**:
    *   Action: Standard add (append to existing list).
5.  **No more #1 updates**:
    *   Action: Disable this feature permanently (persisted in config).
    *   Config Key: `rank_alert_enabled` (default `true` or based on existence).

## Implementation Steps

### 1. Configuration Check
*   [ ] Verify `config.json` has `rank_alert_enabled`. If missing, default to `true`.
*   [ ] Add endpoint or logic to update this config value via UI.

### 2. Frontend Logic (`frequency_explorer.html`)
*   [ ] **State Tracking**: Add `let currentRank1 = null;` variable.
*   [ ] **Detection**:
    *   Inside `fetchData()` or `renderTable()`.
    *   After sorting leaders, identify `newRank1`.
    *   Compare `newRank1.key` (Product+Strategy) with `currentRank1`.
    *   **Auto-Action Check**: Before showing alert, check `config.rank_alert_default_action`. If set, execute immediately.
    *   If changed (and `currentRank1` was not null), trigger Alert.
    *   Update `currentRank1`.
*   [ ] **Alert Component**:
    *   Insert HTML for the alert box (hidden by default).
    *   Add CSS for visibility/styling.
    *   Include "Repeat" checkbox.

### 3. Backend Support (`trade_viewer_api.py`)
*   [ ] **Grid Update**: Ensure `POST /api/grid_live` can handle an "overwrite" flag or simply accept a full list to replace the current file content.
*   [ ] **Config Update**: Ensure `POST /api/config` works for saving the new preference `rank_alert_default_action`.
