[V20260210_1223] Fix Smart Target Activation Logic
**Current Status**: COMPLETED by Claude Code (2026-02-10 12:23)
**Objective**: Fix the `smart_target_activate` logic in `trade_viewer_api.py` which was incorrectly writing metadata to `grid_live.json`, leading to monitoring failures.

## 1. Issue Analysis
The user reported that despite a "Sell" bias being detected, no product/strategy was identified for the grid.
Inspection of `grid_live.json` showed the correct structure for manually added items (using `model` and `group`), but the API logic for AI activation (added in V20260209_1900) was attempting to use `strategy` as the key and boolean flags for buy/sell, which is inconsistent with the `grid_live.json` schema.

**Incorrect Schema used by API:**
```json
{ "strategy": "...", "buy": true, "sell": false }
```

**Correct Schema required by Monitor:**
```json
{ "model": "...", "metric": "buy_net", "group": "..." }
```

## 2. Solution Implemented
Updated the `smart_target_activate` function in `trade_viewer_api.py`:
1.  **Key Correction**: Changed logic to look for and write `model` instead of `strategy`.
2.  **Format Alignment**: Updated the new entry structure to include `metric`, `source`: 'ai_target', `manual`: false, and `group` (defaulting to product name).
3.  **Update Logic**: Ensured that updates to existing entries also respect the correct field names (`metric` instead of boolean flags).

    *   [x] Fixed `smart_target_activate` to use `model`, `group`, and `metric` keys consistent with `grid_live.json`.
