# Plan: Fix Strategy List "unknown" Issue

This plan addresses the issue where the P&L Graph UI shows "unknown" for strategy names. 
The root cause is that trade JSON files do not contain the strategy name fields (`script_name` or `source_strategy`), and the summary generator relies on these fields.

## 1. Requirement Analysis
- UI currently shows "unknown" strategies.
- `_summary_net.json` contains "unknown" keys.
- Trade JSON files (e.g., `breakout_R_Rev_...json`) lack internal strategy identification fields.
- Goal: Ensure strategy names are correctly captured and displayed.

## 2. Plan of Approach
1.  **Tagging**: All changes will be tagged with version **V20260122_1000**.
2.  **Update `common.py`**:
    - Modify `BaseBreakoutStrategy._save_trade_json` to include `script_name` and `source_strategy` in the JSON payload.
    - This ensures all future trades are self-identifying.
3.  **Update `summary_net_generator.py`**:
    - Enhance the strategy name extraction logic.
    - If the JSON content lacks a strategy name, attempt to extract it from the filename.
    - Heuristic: The strategy is the part of the filename before the product name.
4.  **Update Version**: Update `Constants.py` to `V20260122_1000`.
