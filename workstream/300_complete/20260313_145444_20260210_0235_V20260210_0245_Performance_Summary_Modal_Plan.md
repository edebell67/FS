# Implementation Plan: Performance Summary Breakdown Modal

This plan outlines the addition of a "Performance Summary" modal to the `strategy_performance.html` dashboard. This breakdown provides aggregated insights by Category, Window Size, and TP:SL configuration in a pop-up window.

## 1. Requirements
- Add a new "📊 Performance Summary" button to the control header.
- On click, open a pop-up modal displaying performance aggregations.
- **Data Breakdown Dimensions**:
    - **Strategy Category** (Breakout, Breakout_R, Breakout_Rev, Breakout_R_Rev).
    - **Window Size** (2, 3, 4).
    - **TP:SL Configuration** (e.g., 10.0/20.0).
- **Metrics per Card**:
    - #Buys / #Sells counts.
    - Buy Net / Sell Net performance.
    - Total Net for that group.
- Styling must match the premium "Modern Dark" aesthetic of the application.

## 2. Proposed Changes

### 2.1 CSS Integration (`strategy_performance.html`)

- Define `.summary-modal-content` and `.summary-grid-v2` for the card layout inside the modal.
- Reuse dashboard card styles for consistency.

### 2.2 HTML Structure (`strategy_performance.html`)
- Add `<button id="summaryBtn" class="btn-summary" onclick="showPerformanceSummary()">📊 Performance Summary</button>` to the control group.
- Add the modal container `<div id="summaryBreakdownModal" class="modal">...</div>`.

### 2.3 JavaScript Logic (`strategy_performance.html`)
- **Aggregation Library**: Implement logic to group `currentData` by the three dimensions.
- **Parsing**: Use regex to extract window size and TP:SL from names/params.
- **Render**: Create cards dynamically inside the modal.
