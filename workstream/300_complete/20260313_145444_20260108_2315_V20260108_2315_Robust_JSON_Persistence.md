# Plan: Robust JSON Persistence and Error Handling (2026-01-08)

This plan addresses the "Expecting value: line 1 column 1 (char 0)" error by making JSON file writes atomic and improving error handling during state restoration.

## 1. Understanding of Requirements
The user is experiencing errors when `breakout.py` (via `common.py`) attempts to read corrupted or empty JSON trade files. These files are likely created when a process is terminated during a write operation.

## 2. Plan of Approach
1.  **Atomic Writes**: Modify `_save_trade_json` and `_finalize_trade_json` in `common.py` to write trade data to a temporary file first, then use `os.replace()` to move it to the target location. This ensures that the target file is either completely written or unchanged.
2.  **Graceful Restoration**: Enhance `_load_persisted_state` in `common.py` to handle `json.JSONDecodeError` specifically. Instead of a generic exception, it will catch decode errors, log a warning, and skip the problematic file.
3.  **Version Update**: Bump the version in `constants.py`.
