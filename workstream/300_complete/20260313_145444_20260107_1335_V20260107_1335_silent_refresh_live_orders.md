# Plan - Silent Auto-Refresh for Live Orders

This plan describes how to implement a non-disruptive "silent" refresh for the Live Orders page.

## 1. Understanding of Requirements
- **No full-page loading**: During auto-refresh, the page should not clear or show a large spinner.
- **Visual feedback**: A subtle indicator should show when the last update occurred.
- **Explicit vs Background**: Manual refresh (clicking the button) should still show the loading state, while background auto-refresh should be silent.
- **Versioning**: Update to `V20260107_1335`.

## 2. Plan of Approach

### UI Layer (`TradeApps\breakout\live_trades.html`)
1.  **Header Update**:
    - Add a small `<span>` with `id="lastUpdated"` near the title to display the last refresh time.
2.  **Logic Update (`loadLiveTrades`)**:
    - Add `isSilent` parameter (default `false`).
    - Only show the full-page spinner if `isSilent` is `false`.
    - Always update the `lastUpdated` span with the current time upon successful fetch.
3.  **Timer Update (`toggleAutoRefresh`)**:
    - Ensure `setInterval` calls `loadLiveTrades(true)`.

### Versioning
1.  Update `constants.py` and footer of `live_trades.html` to `V20260107_1335`.
