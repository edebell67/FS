# V20260210_0400: Bias-Flip Deactivation & Grid Cleanup Plan

This document outlines the plan to fix the "Activation Leak" where every bias flip adds a new strategy to the grid without removing the old ones, leading to an excessive number of active entries.

## 1. Understanding of Requirements
- The user expects a "Max 2" activations today (one for the start SELL bias, one for the flip to BUY).
- Currently, there are 8-10 entries because multiple flips occurred, and the system is "collecting" instead of "switching".
- Bias flipped many times between 01:47 and 03:30.
- Goal: Ensure that when a bias flip occur, the system deactivates/removes strategies that no longer match the current bias.

## 2. Plan of Approach

### A. Backend Fixes (`trade_viewer_api.py`)
1. **Enhance `/api/deactivate_wrong_bias`**:
   - It must specifically target `source: ai_target` entries and **remove** them if they don't match the `current_bias`.
   - For regular activations, it should set `buy: false` and `sell: false` if they have an `activated_for_bias` tag that contradicts the current state.
2. **Update `/api/grid_live` POST**:
   - Ensure it correctly accepts and saves the `activated_for_bias` field in the model objects.

### B. Frontend Fixes (`ai_picker.html`)
1. **Update `updateData`**:
   - When `biasHasFlipped` is true, immediately call `/api/deactivate_wrong_bias` with the `newBias`.
2. **Update `confirmSmartTarget`**:
   - Pass the `bias` (as `activated_for_bias`) to the `/api/grid_live` POST request so the backend knows which bias this strategy belongs to.

### C. Data Cleanup
1. **Sanitize `grid_live.json`**:
   - Remove the accumulated `ai_target` entries from today that do not match the current bias.
