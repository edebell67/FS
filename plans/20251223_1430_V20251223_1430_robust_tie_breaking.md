# Plan: Robust Fix for Tie-Breaking in Auto-Activation (V20251223_1430)

## 1. Understanding of Requirements
The previous tie-breaking fix was not effective, and multiple strategies are still being activated when they share the same top P&L. The goal is to implement a more robust and explicit tie-breaking mechanism to ensure only `top_n` strategies are activated.

## 2. Plan of Approach
1.  **Update Configuration**: Increment the version number in `constants.py`.
2.  **Modify `common.py`**:
    *   In `_perform_auto_activation_check`, replace the simple sorting logic with a more explicit tie-breaking block of code for both `net` and `alt` candidates. This new logic will handle ties for the top P&L spot by sorting the tied strategies by name and picking the first one(s) up to the `slots_to_fill` limit.
3.  **Update Documentation**: Document the more robust fix in `CHANGELOG.md`.
4.  **Verify**: Confirm that only one strategy is active in `activations.json` after the change.
