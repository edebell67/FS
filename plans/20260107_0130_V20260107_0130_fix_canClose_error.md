# Plan - Fix canClose is not defined error in live_trades.html

This plan addresses the "Error: canClose is not defined" issue reported by the user on the Live Orders screen.

## 1. Understanding of Requirements
The user is seeing a JavaScript error on `live_trades.html` because the variable `canClose` is used in a template literal but is never defined or passed to the function. Additionally, `isVirtual` is also used without being defined.

## 2. Plan of Approach
1.  **Modify `live_trades.html`**:
    *   Update the `createTableSection` function to accept a 4th parameter `canClose`.
    *   In the `createTableSection` loop, use `t._is_virtual` to determine if a trade is virtual when calling `closeTrade`.
    *   Update the version string in the footer.
2.  **Modify `constants.py`**:
    *   Update the `VERSION` constant to reflect the new version.

## 3. List of Changes

### `TradeApps\breakout\live_trades.html`
- [x] Add `canClose` as a 4th parameter to `createTableSection(title, trades, color, canClose)`.
- [x] Update the `closeTrade` call inside the loop to: `closeTrade('${t.filename}', ${t._is_virtual || false})`.
- [x] Update version in footer to `v2026.01.07.0130`.

### `TradeApps\breakout\constants.py`
- [x] Update `VERSION` to `V20260107_0130`.

## 4. Verification Plan
- [ ] Verify that the "canClose is not defined" error is gone.
- [ ] Verify that the "Close" buttons appear only for sections where `canClose` is `true` (Open Positions and Virtual Trades).
- [ ] Verify that clicking "Close" passes the correct `is_virtual` flag to the API.
