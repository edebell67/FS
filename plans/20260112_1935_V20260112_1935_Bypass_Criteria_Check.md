# Plan for Bypass Criteria Check Feature

## Goal
Implement a `bypass_criteria_check` feature that allows forcing execution of strategies based solely on their Activation status and Profitability Guard, ignoring standard price action criteria. This is intended for debugging or forcing trades.

## Requirements
1.  Add `bypass_criteria_check` (boolean) to `config.json` (default: false).
2.  Expose this setting in the "Guards & Safety" section of the Configuration UI (`trade_viewer.html`).
3.  Modify `common.py` trading logic (`process_new_tick`) to checks this flag.
4.  If Flag is ON:
    *   Bypass `check_and_enter` (standard criteria).
    *   Check if strategy is ACTIVE (via `activations.json`).
    *   Check if Profitability Guard passes (via `_is_profitability_guard_passed`).
    *   If both pass, enter trade immediately.
5.  If Flag is OFF:
    *   Proceed as normal (Price Criteria -> Signal -> Enter Trade).
