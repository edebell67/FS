# Plan: Strict Date Filtering for Multi-Chart V3

Date: 2026-01-25
Version: V20260125_2245

## 1. Objective
Fix the "Data Bleed" issue where the Multi-Chart Grid displays data from future dates (e.g., Jan 23) when a historical date (e.g., Jan 20) is selected. This requires implementing a strict date range filter in both the Backend and Frontend.

## 2. Plan of Approach
1.  **Modify API (`api_server_sql\main.py`)**:
    *   Add support for `created_lt` (Less Than) query parameter.
    *   Inject `created < '{created_lt}'` into the SQL WHERE clause logic.
2.  **Modify Frontend (`multi_chart_v3.js`)**:
    *   Update `fetchData` to send both `created_gt` (start of day) and `created_lt` (end of day).
    *   Implement strict client-side filtering to drop any records that fall outside the selected 24h window.
    *   Force `datasetEarliestTime` and `datasetLatestTime` to be exactly the start and end of the selected trade date.
3.  **Update Versioning**:
    *   Update `TradeApps\breakout\fs\constants.py`.
