# Plan: Opposing Trade Generation in `trg_ExportPendingTradeToCSV_CombinedClosed`

**Version:** 64.2.21
**Date:** 2025-11-09

## 1. Understanding of Requirements

The goal is to modify the `trg_ExportPendingTradeToCSV_CombinedClosed` trigger. Instead of generating a single JSON file to close a trade, it must now generate two:
1.  **Closing Trade JSON:** The first file will represent the action needed to close the trade, using the signal provided by the `fn_expand_trade_legs` function.
2.  **Opposing Trade JSON:** The second file will represent a new trade being opened in the opposite direction of the first. The signal for this trade will be the inverse of the closing trade's signal (BUY becomes SELL, SELL becomes BUY).

Both files must be named appropriately to distinguish them.

## 2. Plan of Approach

The core of the change will be within the `CURSOR` loop of the trigger. For each trade leg processed by the cursor, I will perform the JSON generation and logging steps twice.

1.  **Generate Closing Trade (First File):** I will keep the existing logic for generating the first JSON file. I will modify the filename to include a `_close` suffix to clearly identify its purpose.
2.  **Generate Opposing Trade (Second File):**
    *   Inside the same loop, I will determine the "flipped" signal (e.g., if the closing signal is 'SELL', the flipped signal will be 'BUY').
    *   I will then generate a second JSON payload using this flipped signal for the `action`.
    *   A second file will be written using a different name, for instance, with an `_open` suffix, to represent the new opposing trade.
3.  **Logging:** I will ensure that an entry is made in the `dbo.tbl_rt_trades` table for *each* JSON file generated, accurately logging the details of both the closing and the new opposing trade.

## 3. List of Changes

*   **File:** `db_scripts/dbo.trg_ExportPendingTradeToCSV_CombinedClosed.Trigger.sql`
    *   [ ] Inside the `export_cursor` loop, modify the filename generation for the **first (closing) trade** to include a `_close` suffix (e.g., `..._pending_close_tradeable_close.json`).
    *   [ ] After the first `xp_cmdshell` call, add logic to determine the opposing signal (e.g., `IF @signal = 'BUY' SET @flipped_signal = 'SELL' ELSE SET @flipped_signal = 'BUY'`).

    *   [ ] Add a second `INSERT` statement into `dbo.tbl_rt_trades` to log the creation of the opposing trade's JSON file, ensuring the correct flipped signal is recorded.
