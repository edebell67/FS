# Gemini Coder - Plan for Agent 03: Positive Peak Reverser Strategy

This document outlines the plan to implement the new **"Positive Peak Reverser"** strategy for `agent 03`, based on the hypothesis that 'person' participants reaching Rank #1 with positive realized P/L are likely to experience a reversal in their subsequent trades.

## 1. Understanding of Requirements

The goal is to create a new agent (`agent 03`) that acts as a contrarian to 'person' participants who achieve the #1 rank while having a positive `realized_pl`.

*   **Target Identification:** Identify a 'person' participant who becomes Rank #1 AND has a positive `realized_pl` at that moment.
*   **Entry Condition:** When such a participant is identified, `agent 03` will open a trade in the **opposite direction** of the tracked participant's current open position.
*   **Exit Condition:** `agent 03` will close its open position when the tracked 'person' participant is no longer Rank #1.

## 2. Plan of Approach

1.  **Constants & Configuration (`config.py`):**
    *   Add `agent 03` to `RL_AGENT_IDS`.
    *   Add a new global flag `EXPORT_AGENT_03_TRADES_DEFAULT`.

2.  **Participant Creation (`trader_game.py`):**
    *   Modify the participant generation logic to create `agent 03` and assign it the `'positive_peak_reverser'` strategy.
    *   Update the `simulation_state` dictionary to include `export_agent_03_trades`.
    *   Add a new API endpoint `/admin/set_export_agent_03_trades/<string:value>`.

3.  **Strategy Implementation (`market_simulator.py`):**
    *   Create a new function `sync_agent_positive_peak_reverser(agent_state, leaderboard_data, updated_participants_states, current_bid, current_ask, global_state)`.
    *   This function will manage the state of `agent 03` (e.g., `tracking_leader_id`, `leader_trade_type`, `agent_open_trade_id`).
    *   It will iterate through the `leaderboard_data` to find the current #1 'person' participant.
    *   If a new #1 'person' with positive `realized_pl` is found:
        *   If `agent 03` has an open position, it will close it.
        *   It will then open a new trade in the opposite direction of the new #1 person's current open position (if any).
    *   If `agent 03` is tracking a leader who is no longer #1, it will close its current position.
    *   Integrate this new function into the main `update_market_prices` loop.
