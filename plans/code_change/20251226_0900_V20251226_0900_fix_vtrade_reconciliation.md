# Implementation Plan - Fix Virtual Trade Reconciliation (V20251226_0900)

## 1. Goal Description
The objective is to fix the virtual trade (V-Trade) reconciliation mechanism. Currently, reconciliation is skipped if no performance data (closed trades) is found for the current date, which can lead to multiple "orphan" virtual trades staying open incorrectly. We need to ensure reconciliation always runs and correctly handles the transition between different trading products and directions in both live and simulation modes.

## 2. Reconciliation Mechanics (Single Trade Enforcement)
To prevent the build-up of multiple open trades, the refactored logic will:
1.  **Identify Top Performer**: Determine the single "best" product/strategy per direction (LONG/SHORT) from today's performance data.
2.  **Global Scan**: Scan the *entire* virtual trade directory for all files where `"status": "OPEN"`.
3.  **Strict Filtering**: 
    - For every open trade, check if it matches the "Top Performer".
    - If it does **not** match, it is immediately closed with `TOP_PRODUCT_CHANGED`.
    - If no performance data exists (e.g., at the start of the day), **all** open trades are closed because none can match the (empty) top performer.
4.  **Threshold Enforcement**: Even if a trade matches the top product, it is closed if its PnL is below the `guard_threshold_y`.

## 3. User Review Required
> [!IMPORTANT]
> This change ensures that if no strategy is performing above the `guard_threshold_y`, ALL virtual trades for that direction will be closed. This is intended behavior to prevent trading during poor performance periods.
