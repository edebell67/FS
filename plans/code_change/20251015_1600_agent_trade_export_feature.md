# Agent Trade Export Feature (2025-10-15)

## 1. Request Overview

The user requested a new feature to selectively export trades generated by `agent 01` and `agent 02` as JSON files. This export should be controlled by new checkboxes in the UI.

**Key Requirements:**
*   Trades to be exported as JSON files.
*   Output folder: `C:\users\edebe\eds\trades_rt\orders\`
*   JSON format to match the input expected by `tws_monitor_and_trade2.py` (i.e., `{"symbol": "...", "secType": "CASH", ...}`).
*   Filename format: `GT_[participant_id]_[trade_id]_tradeable.json`.
*   Two new checkboxes in the UI: "Export Agent 01 Trades" and "Export Agent 02 Trades".
*   If a checkbox is selected, trades generated by that agent are exported.

            export_json = {

            file_path = os.path.join(AGENT_TRADE_EXPORT_FOLDER, filename)
            try:
                with open(file_path, 'w') as f:
                    json.dump(export_json, f, indent=4)

                    'participant_id': participant['id'],
                    'trade_id': trade_id,
                    'file': filename,
                    'timestamp': time.time()

                print(f"Exported agent trade to: {file_path}")
            except Exception as e:
                print(f"Error exporting agent trade to JSON for {participant['id']}: {e}")

## 3. Circular Import Resolution

A circular import issue between `trader_game.py` and `trading_engine.py` was identified and resolved by:
*   Removing the direct import of `trader_game` from `trading_engine.py`.
*   Modifying the `execute_trade` function in `trading_engine.py` to accept the export flags as explicit parameters.
*   Updating all calls to `execute_trade` in `market_simulator.py` to pass these flags from the `global_state` (which is the `simulation_state`).
