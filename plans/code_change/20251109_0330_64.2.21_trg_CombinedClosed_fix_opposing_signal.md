# Plan: Fix Opposing Signal in `trg_ExportPendingTradeToCSV_CombinedClosed`

**Version:** 64.2.21
**Date:** 2025-11-09

## 1. Understanding of Requirements

The goal is to fix a bug in the `trg_ExportPendingTradeToCSV_CombinedClosed` trigger. Currently, when a trade is closed, the generated JSON file for the closing action contains the same signal as the original trade, which is incorrect.

The trigger must be modified to ensure the `action` in the JSON file is always the **opposite** of the original trade's signal.
*   Closing a `BUY` trade must generate a `SELL` action.
*   Closing a `SELL` trade must generate a `BUY` action.

This is a bug fix, and only one JSON file should be generated per trade leg.

## 2. Plan of Approach

The fix will be implemented inside the `CURSOR` loop of the trigger. The `export_signal` returned from the `fn_expand_trade_legs` function appears to be the source of the issue, as it is not the correct opposing signal. I will add logic to "flip" this signal before it is used.

1.  **Determine Correct Closing Action:** Inside the loop, after fetching the `@signal` from the cursor, I will add logic to determine the correct opposing signal.
2.  **Use Correct Signal:** I will then use this corrected, opposing signal to generate the JSON file's `action`, the filename, and the log entry in `dbo.tbl_rt_trades`.

## 3. List of Changes

*   **File:** `db_scripts/dbo.trg_ExportPendingTradeToCSV_CombinedClosed.Trigger.sql`
    *   [ ] Inside the `export_cursor` loop, declare a new variable `@closing_action NVARCHAR(10)`.
    *   [ ] Add logic to set `@closing_action` to the opposite of the `@signal` variable (e.g., `IF @signal = 'BUY' SET @closing_action = 'SELL' ELSE SET @closing_action = 'BUY'`).
    *   [ ] Update the `CONCAT` statement for the `@FileName` to use `@closing_action` instead of `@signal`.
    *   [ ] Update the JSON generation logic to use `@closing_action` for the `action` field.
    *   [ ] Update the `INSERT` statement for `dbo.tbl_rt_trades` to log `@closing_action` in the `signal` column.
    *   [ ] Add a comment explaining the signal is being flipped to fix the bug.
