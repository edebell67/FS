# Plan: `sp_001_zone_distribution_trade` Flip Trade Variable

**Version:** 64.2.20
**Date:** 2025-11-09

## 1. Understanding of Requirements

The goal is to replace the hardcoded `0` for the `flip_trade` column in the final `INSERT` statement of `sp_001_zone_distribution_trade`. A new variable, `@flip_trade`, will be introduced, initialized to `1`, and used in its place. This provides a single, clear location at the top of the script to configure the `flip_trade` behavior for all trades generated by this procedure.

## 2. Plan of Approach

1.  **Declare Variable:** I will declare a new `BIT` variable named `@flip_trade` and set its default value to `1`. This declaration will be placed in the configuration section (Section 2) of the stored procedure for easy access and visibility.
2.  **Update INSERT Statement:** I will modify the `INSERT dbo.combined_trades_open` statement in Section 6. The hardcoded `0` currently used for the `flip_trade` column will be replaced with the new `@flip_trade` variable.

## 3. List of Changes

*   **File:** `db_scripts/dbo.sp_001_zone_distribution_trade.StoredProcedure.sql`
    *   [ ] In the configuration section (Section 2), add the declaration: `DECLARE @flip_trade BIT = 1;`.
