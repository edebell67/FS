# Gemini Coder - Plan for SQL Server to PostgreSQL Data Synchronization

## 1. Understanding of Requirements

The goal is to establish a data synchronization process for three key data entities from SQL Server to PostgreSQL:
1.  **`combined_trades_closed_arc`**: A table containing archived closed trade data.
2.  **`zone_distribution_snapshots`**: A table storing snapshots of zone distribution data.
3.  **`vwCombined_trades_open`**: A SQL Server view that needs to be materialized into a new PostgreSQL table named `Combined_trades_open_snapshot`, with data refreshed every minute.

The synchronization process should be asynchronous and triggered when data appears in SQL Server (for the first two tables) or on a minute-by-minute schedule (for the snapshot table). PostgreSQL credentials are available in `C:\Users\edebe\eds\api_server_pg\.env`. SQL Server table/view definitions are in `C:\Users\edebe\eds\db_scripts`.

## 2. Plan of Approach

This plan will involve several steps:
*   **Schema Translation**: Convert SQL Server DDL to PostgreSQL DDL for the target tables.
*   **PostgreSQL Table Creation**: Create the necessary tables in the PostgreSQL database.
*   **SQL Server Linked Server Setup**: Establish a SQL Server Linked Server to connect to the PostgreSQL database.
*   **Data Synchronization (T-SQL)**: Develop T-SQL scripts (likely within SQL Server Stored Procedures) to perform data transfer using the Linked Server.
*   **Scheduling**: Implement a scheduling mechanism using SQL Server Agent for the T-SQL synchronization jobs.
