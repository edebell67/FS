### Development Plan: Quant-Trading Omni Dashboard Prototype

The goal is to create a functional, visually modern, and responsive web application prototype that fulfills the core requirements of the design brief.

**1. Technology Stack:**

*   **Frontend Framework:** React with TypeScript for a robust, type-safe, and component-driven architecture.
*   **Build Tool:** Vite for fast development and optimized builds.
*   **Styling:** Tailwind CSS to rapidly implement the modern, clean, and responsive design. A dark/light theme switcher will be included.
*   **Charting:** Recharts for creating interactive and visually appealing charts (sparklines, heatmaps).
*   **State Management:** Zustand for a simple and efficient global state management solution, ideal for handling the watch-list and global filters.
*   **Backend:** Mock data will be created in JSON files to simulate API responses, allowing for full frontend development without a live backend.

**2. Project Structure & Scaffolding:**

I will begin by scaffolding a new React project using Vite. The application will be structured into logical, reusable components (e.g., `MetricCard`, `PortfolioTile`, `CorrelationMatrix`, `NotificationPanel`) to ensure the codebase is modular and extensible, as per the design brief.

**3. Implementation Approach:**

The development will proceed in the following phases:

*   **Phase 1: Core Layout & Data Mocking**
    *   Create mock JSON data files that adhere to a defined schema for portfolios, strategies, trades, and top-level metrics.
    *   Implement the main dashboard layout, including the top-level metrics hero strip, the multi-perspective switcher, and placeholders for the main content areas.

*   **Phase 2: Component Development & Visualization**
    *   Build the primary UI components for displaying portfolios, strategies, and trades.
    *   Implement the "Top 5 / Bottom 5" comparative tiles using the mock data.
    *   Create the `CorrelationMatrix` component using a Recharts heatmap.
    *   Add lifecycle "age" tags and other visual indicators as specified.

*   **Phase 3: Interactivity & State Management**
    *   Implement the client-side state management for the watch-list feature using Zustand.
    *   Develop the drill-down navigation logic to move from a global view to individual items.
    *   Create the slide-in notification panel and the placeholder trade execution modal.

*   **Phase 4: Finalization & Theming**
    *   Implement the dark and light themes using CSS variables and Tailwind CSS.
    *   Ensure the application is fully responsive, adapting seamlessly from mobile to desktop viewports.
    *   Add skeleton loaders to simulate a high-performance loading experience.

**4. Deliverable:**

The final deliverable will be a functional web application prototype that you can run and interact with. It will demonstrate the key UI features, interactions, and the visual design outlined in the brief. While I cannot produce Figma or QML files, this HTML/React prototype will serve as a high-fidelity, code-based implementation of your vision.
