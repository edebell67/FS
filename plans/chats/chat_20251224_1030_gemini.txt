  The V-Trade Lifecycle

  Think of a V-Trade as a "blueprint" or a "candidate" for a live trade. Its purpose is to always represent the most
  promising trade idea for both BUY and SELL directions, based on real-time performance.

   1. Identifying the "Best Idea": On each cycle (approx. every 60 seconds), the system analyzes the performance of all
      strategies. It identifies the single best-performing product for the top BUY strategy and, separately, the single
      best-performing product for the top SELL strategy.

   2. Creating/Maintaining the V-Trade:
       * Creation: If there's a top-performing BUY product (e.g., GBP) and no open BUY V-Trade exists, the system
         creates one. A new JSON file, like vt_..._GBP_....json, is generated in the .../virtual/ directory with status:
         'OPEN'.
       * Persistence: This GBP BUY V-Trade will remain OPEN as long as GBP is the top-performing product for the BUY
         direction. The V-Trade doesn't care if the strategy that made it the top performer changes, only that the
         product remains the best.
       * Closure & Replacement: If, on a later cycle, EUR becomes the new top-performing product for the BUY direction,
         the system will automatically close the GBP V-Trade (by setting its status to CLOSED in the JSON file) and
         instantly create a new V-Trade for EUR.

  This ensures there is always a "candidate" trade ready for both the best BUY and best SELL opportunities.

  Bridging the Gap: V-Trade to L-Trade Promotion

  This is the key to creating L-Trades from V-Trades. The logic I implemented previously had a placeholder here, which I
  will now detail.

  To make this work, the function responsible for creating live trade orders, _create_tradeable_json, needs to be
  callable from the V-Trade logic. Here is how it will function:

   1. Refactor for Reusability: I will refactor the _create_tradeable_json logic out of the BaseBreakoutStrategy class
      and into a new, standalone helper function in common.py. This new function, let's call it _create_l_trade_order,
      will take trade details (like product, direction, etc.) as arguments.

   2. Promotion Check: On every cycle, the _manage_virtual_trades function will perform a promotion check for each open
      V-Trade (both BUY and SELL). The conditions for promotion are:
       * The V-Trade's is_live_trade flag is true. (This is configurable in config.json via the
         virtual_trade_live_by_default setting).
       * The global open_live_trades_count is 0 (meaning there are no other active L-Trades).

   4. L-Trade Generation: The _create_l_trade_order function will then generate the TWS-compatible JSON order file and
