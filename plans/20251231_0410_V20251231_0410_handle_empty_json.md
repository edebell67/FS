# Plan: Handle Empty JSON Files in State Persistence

This document outlines the plan to handle 0-byte or corrupted JSON files gracefully during the script restart/persistence process.

## 1. Understanding of Requirements

*   The user is seeing errors when the script tries to load empty (0-byte) JSON files during the `_load_persisted_state` process.
*   The script should skip these files silently or with a minimal warning instead of throwing a decoding error message.

## 2. Plan of Approach

1.  **Modify `common.py`**:
    *   In `_load_persisted_state`, add a check to verify the file size before attempting to `json.load`.
    *   If the file is empty, skip it with a simple debug log.
    *   Update the error handling to be more specific (distinguish between file access errors and JSON decoding errors).
2.  **Update Version**:
    *   Update `VERSION` in `constants.py` to `V20251231_0410`.
