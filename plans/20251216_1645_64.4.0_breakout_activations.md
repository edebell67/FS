# Implementation of Activation-Based Order Generation for Breakout Strategies

**Date:** 2025-12-16 16:45
**Version:** 64.4.0 (Estimated based on context)
**Description:** Enable activation checkboxes in Trade Viewer to control order generation for Net and Alt strategies, with specific logic to flip direction for Alt orders.

## 1. Understanding of Requirements

The user requested the ability to activate/deactivate "Net" and "Alt" order generation for each breakout strategy via the Trade Viewer UI. A key requirement was that "Alt" orders should have their direction "flipped" (e.g., a Long signal triggers a Sell order) to support reverse trading logic. Generated order files must be saved to the path specified in `config.json` (`send_json_files`).

## 2. Plan of Approach

1.  **API & UI (Completed previously):**
    *   Add endpoints to save/load activation status to `activations.json`.
    *   Add checkboxes to UI keying off `{appname}_{direction}_{mode}`.

2.  **Strategy Scripts (`breakout.py`, `breakout_R.py`, `breakout_Rev.py`, `breakout_R_Rev.py`):**
    *   Implement `_is_active(direction, mode)` method to read from `activations.json`.
    *   Update `_create_tradeable_json` to:
        *   Accept `mode` and `is_close` arguments.
        *   Flip `base_action` if `mode == 'alt'`.
        *   Handle Open/Close inverted logic.
        *   Save to `send_json_files` path from config.
    *   In `check_and_enter`:
        *   Check `_is_active` for 'net' and 'alt'.
        *   Trigger `_create_tradeable_json` if active.
    *   In `close_trade` (and equivalents):
        *   Trigger closing orders if entry orders were sent.
