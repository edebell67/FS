# Plan: Auto Refresh Follow Mode and Boundary Sync [V20260126_0400]

This plan ensures that the "Auto Refresh" feature in both Multi-Chart dashboards (V2 and V3) actually keeps the charts up to date by moving the playback point to the latest "now" time when new data is fetched.

## 1. Requirements
*   In "Auto Refresh" mode, if the user is not actively playing a recording, the charts should automatically jump to the latest available time point (the new "end of day" which is "now" for today's date).
*   Sync the time slider boundaries in V2 to match V3 (always show the full day 00:00 to 23:59 or 00:00 to Now).
*   Fix V2's data fetching URL to include the same end-of-day constraint as V3.

## 2. Plan of Approach

### A. Versioning
*   [ ] Update `constants.py` to `V20260126_0400`.

### B. Multi-Chart V3 JS (`multi_chart_v3.js`)

    *   Action: Set `playbackTime = boundaries.endOfDay;`
    *   This ensures that every auto-refresh "pushes" the view to the latest data.
