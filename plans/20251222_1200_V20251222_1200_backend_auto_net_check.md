# 20251222_1200_V20251222_1200_Backend_Auto_Net_Check

## 1. Understanding of Requirements
The user reported that "Auto Net Check" is activated in the configuration but is not triggering any net checks. Investigation revealed that this feature was implemented solely in the frontend (`trade_viewer.html`), requiring the dashboard to be open to function.
The goal is to migrate this logic to the Python backend (`common.py`) so that strategies can automatically activate themselves based on performance criteria (Net P&L > Threshold) without needing the UI to be running.

## 2. Plan of Approach
1.  **Modify `common.py`**:
    *   Implement a new function `_perform_auto_activation_check(target_script_names)`.
    *   This function will:
        *   Read `config.json` to verify `auto_net_check` or `auto_alt_net_check` is enabled.
        *   Read `activations.json` to get current state.
        *   Scan the `json/live/{date}` directory for *today's* closed trades.
        *   Filter trades belonging to the current process's strategies (`target_script_names`).
        *   Aggregate P&L (Net and Alt) for each strategy/direction.
        *   Compare against `auto_net_threshold` / `auto_alt_threshold`.
        *   If criteria met and currently inactive, update `activations.json`.
    *   Integrate this check into the main `run_multiwindow` loop.
    *   Add a timer to ensure it only runs periodically (e.g., every 60 seconds) to minimize disk I/O.

2.  **Scope**:
    *   This change affects `common.py`, which is shared by all breakout strategies (`breakout.py`, `breakout_R.py`, etc.).
    *   Each running strategy script will responsibly check and activate its own specific configurations.
