# Plan: Live Grid Trading - L-Trade Process Integration

Date: 2026-01-25
Version: V20260126_0900

## 1. Objective
Enable "Live Trading" for specific models directly from Multi-Chart V3. When a card is marked "Live", a backend monitor will watch SQL Server for new trades of that model and generate corresponding L-trades (TWS JSON orders).

## 2. Requirements Check
- [x] UI: "Live" toggle button on each chart card.
- [x] UI: Visual "LIVE" pulse indicator.
- [x] Persistence: Save/Load "Live" status of models.
- [x] Backend Monitor: Watch `dbo.combined_trades_open`.
- [x] Backend Monitor: Watch `dbo.combined_trades_closed`.
- [x] L-Trade Integration: Send JSON orders to TWS.
- [x] Logic: Only trade models in "positive territory".

## 3. Plan of Approach

### Phase 1: Frontend & API (Today)
1.  **Update `multi_chart_v3.html`**:
    *   Add "Live" button icon/style.
2.  **Update `multi_chart_v3.js`**:
    *   Add `isLive` flag to `activeOverlays`.
    *   Implement `toggleLive(groupName)`.
    *   Update `createGroupCard` to show the toggle and status.
3.  **Update `api_server_sql/main.py`** (or `trade_viewer_api.py`):
    *   Since Multi-Chart uses `8001` (SQL API), I'll add the persistence endpoints there.
    *   Create `grid_live.json` in root to store `[model, product]` pairs that are live.

### Phase 2: Live Monitor Service
1.  **Create `grid_live_monitor.py`**:
    *   Use `db_utils.py` for SQL connectivity.
    *   Use `common.py` for L-trade order generation (`_create_l_trade_order`).
    *   Main Loop:
        *   Fetch live models from `grid_live.json`.
        *   Check for open trades in `dbo.combined_trades_open` that match live models.
        *   Validate "Positive Territory" (using `vw_top_performer` logic or simple P&L sum).
        *   If new trade + positive -> Create OPEN L-Trade.
        *   Monitor `dbo.combined_trades_closed` for closure of active L-trades.
        *   If closed -> Create CLOSE L-Trade.
