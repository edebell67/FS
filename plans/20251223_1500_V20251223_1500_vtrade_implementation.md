# Plan: V-Trade Implementation (V20251223_1530)

## 1. Understanding of Requirements
Implement a new "Virtual Trade" (V-Trade) system that mirrors the top-performing strategy's product and direction. V-Trades are stored in a separate directory, can be promoted to live trades, and have their lifecycle managed based on changes in the top-performing product for a given direction (BUY/SELL). The final piece is to enable promotion of V-Trades to L-Trades by integrating the L-Trade order creation logic.

## 2. Plan of Approach
1.  **Refactor L-Trade Order Creation**: Extract core L-Trade order creation logic into a new reusable helper function.
2.  **Integrate L-Trade Promotion**: Update `_manage_virtual_trades` to call this new helper function for V-Trade promotion.
3.  **Update Configuration and Versioning**: Increment version number and document changes.
