# Plan: Dynamic Configuration Reloading

## 1. Requirement
The user requested that changes to "Profitability Guard" settings take effect without restarting the underlying trading applications. Currently, these settings are loaded only once at startup.

## 2. Solution
We will modify the trade execution logic to reload the configuration from `config.json` immediately before processing live orders. This ensures that the latest `profitability_guard` parameters and `max_live_trades` limits are applied to every new trade opportunity.

## 3. List of Changes

- **`TradeApps/breakout/common.py`**:
    - [ ] Modify `BaseBreakoutStrategy._handle_live_orders`:
        - Add a call to `_load_config()` at the start of the method.
        - Update `self.profitability_guard_config` with the fresh values.
        - Read `max_live_trades` from the fresh config instead of the global constant.
- **`TradeApps/breakout/constants.py`**:
    - [ ] Update `VERSION` to `V20251222_0045`.

## 4. Verification
- Verify that changing `profitability_guard` in `config.json` (e.g., toggling `mode` or `threshold_y`) is reflected in the logs for the next trade attempt without restarting the script.
