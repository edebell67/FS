# Breakout FS Changes Summary (2026-02-17)

This document summarizes the fixes made in the latest session across Multi-Chart, workflow automation, bias enforcement, and live-trade safety controls.

## 1) Multi-Chart Presets UX and Delete Behavior

### What was changed
- Restored normal preset control size (single-line preset dropdown).
- Added compact preset selector panel (filter icon) with checkbox list for multi-select.
- Added panel actions:
  - `Delete` (delete selected presets)
  - `Done` (close panel)
- Kept normal preset loading from main dropdown.

### Files
- `TradeApps/breakout/fs/multi_chart.html`
- `TradeApps/breakout/fs/multi_chart_v2.html`
- `TradeApps/breakout/fs/multi_chart_v3.html`
- `TradeApps/breakout/fs/multi_chart.js`
- `TradeApps/breakout/fs/multi_chart_v2.js`
- `TradeApps/breakout/fs/multi_chart_v3.js`

### Behavior now
- You can select multiple presets from the selector panel and delete in one operation.
- If nothing is selected in the panel, delete falls back to currently selected dropdown preset.

## 4) Profile Workflow Candidate Pool Expansion

### Problem
- `profile_match_workflow` reads `_top20.json`, which can under-represent scalpers.

### What was changed
- `_top20.json` generation now includes:
  1. Base top 20 by `total_net` (existing behavior)
  2. Plus all additional positive-net scalpers not already in top 20

### File
- `TradeApps/breakout/fs/summary_net_generator.py`

### Behavior now
- Profile workflow (still using `_top20.json`) sees a wider scalper universe.

---

## 5) Market Bias Freshness (No Stale Cache)

### Problem
- `market_bias_latest` in trade files could stay stale due to cached bias lookup.

### What was changed
- Bias lookup in `common.py` was changed to read `_targeted_strategies.json` directly each call.
- No effective in-process cache return path for bias value.

### File
- `TradeApps/breakout/fs/common.py`

### Behavior now
- Entry/exit bias checks and trade-file bias writes use current source file value at call time.

---

## 6) L-Trade Cap Stability (`max_live_trades`)

### Problem
- Multiple open L-trades were observed under same TB group despite `max_live_trades=1`.
- Root cause: cap scope key could drift with mutable `source_group` lifecycle.

### What was changed
- Introduced stable cap identity field `live_cap_group` in open trade records.
- Live-trade count function now prefers `live_cap_group` for cap counting.
- Tradeable order creation path now uses `live_cap_group` first.

### File
- `TradeApps/breakout/fs/common.py`

### Behavior now
- Max-live cap is enforced against stable group identity, reducing cap bypass via group mutation.

---

## 7) Bias History Reliability + Session Support

### Problems
- `bias_history.json` had corruption.
- Session-specific flips (post-archive) were not reliably queryable.

### What was changed
- Added safe history loading with corruption fallback and backup.
- Added atomic JSON writes for bias history.
- Added session metadata file and support:
  - `bias_history_session.json`
- Extended `/api/bias_history`:
  - `POST`: supports `reset_session`, `session_start`
  - `GET`: supports `since_session=true`, returns `start_bias`, `first_flip_at`, `flip_count`
- Seeded live session start to archive boundary:
  - `2026-02-17T13:52:12+00:00`

### File
- `TradeApps/breakout/fs/trade_viewer_api.py`
- `TradeApps/breakout/fs/bias_history_session.json`

---

## 8) Backend Hard Safety Net for Wrong-Bias Open L-Trades

### Problem
- Wrong-bias open L-trades persisted in some runs.

### What was changed
- Added backend worker enforcement in API loop:
  - Sync open trade files to current bias (`market_bias_latest`)
  - Force-create CLOSE orders for open live trades that mismatch current bias
  - Mark trade files with close metadata:
    - `bias_exit_close_sent_at`
    - `bias_exit_target_bias`
    - `bias_exit_close_file`
- Added config gate: this safety net runs only when:
  - `enforce_market_bias_exit = true`

### File
- `TradeApps/breakout/fs/trade_viewer_api.py`

### Behavior now
- Even if a strategy loop misses an exit, backend worker sends close order for wrong-bias open live trade.

---

## 9) New Bias Audit Endpoint

### What was added
- `GET /api/bias_audit?mode=<live|sim>&date=YYYY-MM-DD`

### Returns
- Current bias for mode/date
- Open live trade count
- Mismatch count
- Mismatch list (file, strategy, direction, recorded bias fields)

### File
- `TradeApps/breakout/fs/trade_viewer_api.py`

---

## 10) Runtime Notes

### Restart requirement
After these patches, both must be restarted:
1. Strategy processes (use `common.py` logic)
2. API server (`trade_viewer_api.py` worker + endpoints)

### Current operational expectation
- `enforce_market_bias=true` blocks wrong-direction new L-trade entries.
- `enforce_market_bias_exit=true` triggers bias-exit in strategy loop and backend safety-net close fallback.
- `/api/bias_audit` should quickly show if any open trades are misaligned.
